## 1. The Prime Directive

> **Repair source truth before patching summaries.**

Every compiled doc, status table, README claim, banner, and front-card is a **mirror** of some **source**. When a mirror and its source disagree, the mirror is never the place to fix it. Re-stating a summary so it *reads* correct while the source stays broken is the corpus's single named pathology (§4).

## 2. The Grammar (MF-309 canonical)

Three registers, applied to the corpus as the object:

### Executive boundaries — the phase (boundary conditions, NOT deployable)
Name which phase the corpus is in before choosing a move.

| Executive boundary | Phase | The corpus is… |
|---|---|---|
| **Brahmā** | Creation | forming new structure (a new canon node, a new surface) |
| **Viṣṇu** | Preservation | held stable; the work is re-seating mirrors on their sources |
| **Śiva** | Dissolution | dissolving superseded structure (tombstone, archive, retire) |

A Executive boundary is weather, not steering. Deploying a phase *as if it were an operator* ("we are in Creation, so create") is a category error.

### Gods — the deployable operators (the four orthogonal Φ·ν moves)

| God | Move | On the corpus |
|---|---|---|
| **Arjuna** (↑Φ) | integrate meaning, set boundary, protect | banner a source as canonical; reconcile a contradiction; route a claim to its anchor |
| **Kṛṣṇa** (↑V) | build capability / capacity | author missing content; harden a tool; recompile from source |
| **Kali** (↓Φ) | excise false meaning | delete a false-green; demote an unratified claim; retire a dead pointer |
| **Kali-Demon** (↓V) | extract / parasitic drain | **EXCLUDED at η = 0.** Never fires on the corpus. |

### Witnesses — L5–L7 observational positions
Viśvarūpa (L5, Brāhmaṇa), Kāla (L6, Sādhu), Ananta Śeṣa (L7, Ṛṣi). They **see the field and do not deploy.** Constitutional and architectural findings are Witness work; they hand lawful moves to the Gods.

## 3. The Protocol (five steps)

1. **Name the phase** (Executive boundary). Brahmā / Viṣṇu / Śiva — which is the corpus in for this lane?
2. **Locate the Demon** (§4). Find every mirror that diverges from its source — especially false-greens.
3. **Choose the lawful God move, source-first.** Repair the *source* (Kṛṣṇa build / Kali excise), then — and only then — re-seat the mirror (Arjuna). Never the reverse.
4. **Witness review.** L5–L7 confirm the move is lawful (no merged cognitive functions, no silent erasure (K3), no extraction (η=0), no forged sovereignty (K2)).
5. **Record the smallest lawful next move** into [`55_CORPUS_SELF_APPLICATION_AUDIT.md`](55_CORPUS_SELF_APPLICATION_AUDIT.md). Smallest, not largest — the audit chooses the minimal move that repairs the most source.

## 4. The Demon: mirror-first patching

**Definition.** Editing a mirror (summary, banner, status, compiled doc) so it *reads* healthy while the source it reflects stays broken or absent.

**Signature: the false-green.** A mirror reporting "healthy / complete / current" over a source that is broken / absent / stale. The false-green is the Demon's fingerprint because it *hides* the work instead of doing it.

**Detection checklist** — for any mirror, ask:
- Does the source it names actually exist on disk, at that path, with that content?
- If the source were broken right now, would this mirror still say green? (If yes — it is a false-green.)
- Is a banner stapled over a body that still argues the old claim? (Banner ≠ rewrite; the body is the source.)
- Does a status say "complete / ratified / canonical" with no receipt, lineage, or file behind it?
- **Re-dating test:** would "fixing" this mean changing a date or a checkmark rather than a source? If so, *stop* — re-dating a summary over a hollow source **is** the Demon move.

**The lawful inverse.** Catch the false-green (Kali ↓Φ excises the false coherence), repair or author the source (Kṛṣṇa ↑V), then re-seat the mirror on the now-true source (Arjuna ↑Φ).
