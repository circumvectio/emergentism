## 1 · The purity criterion (operational test)

**One sentence:** a sentence or file belongs to the purified canon if and only if its
truth or value **stands unchanged when every named venture, organ, product, and site
is deleted from the world** — everything else is either connective tissue in service
of the canon (kept, labeled) or another lane's property (moved, tombstoned).

**Mechanical procedure — two auditors, same verdict:**

1. **Proper-noun underline.** Mark every occurrence of a lane-owned proper noun:
   Skyzai, Circle, RealityFutures, APU.BOT, Agentz, Aureus, OFN, QNTM, PRISM
   (as tokenomics product), AIA engine, noosphere, plus any `02_SKYZAI/` or
   `03_VENTURES/` path. `[S]`-verifiable by grep; no judgment involved.
2. **Substitution test (per marked noun).** Replace the noun with *"a hypothetical
   system implementing the invariants."* If the sentence's argument survives intact,
   the noun was **illustration** → class (c). If the sentence collapses into
   strategy-for-that-product or its truth now depends on that product existing →
   class (d). Binary; disagreements resolved by reading the sentence aloud with the
   substitution in place.
3. **Survival test (unmarked sentences).** Does the claim hold for a mind-independent
   world with no Magnum Opus, no organs, no site? Yes → candidate (a)/(b). No, but it
   serves transmission/illustration/funnel/strategy of those that do → class (c).
4. **Axiom vs. discovery key.** Candidate is an **axiom (a)** if it is a posited
   load-bearing premise of the Weltanschauung (kernel W0–W9 class; marked `[S]/[I]`
   premise, not finding). It is a **discovery (b)** if it is a *find* — something the
   audit wave tried to break and could not: receipt-126/126.1 survivors, the seven
   stones, the second-look survivors. Mechanical key: present in the kernel minimal
   graph or the receipt-126 survivor tables → (b); posited by the owner/framework as
   premise → (a).

**Output classes:**

| Class | Name | Definition | Disposition |
|---|---|---|---|
| (a) | Emergentist axiom | Posited premise; survives the survival test | Untouched; `canon_class: axiom` on next touch |
| (b) | Emergentist discovery | Audit-surviving find; survives the survival test | Untouched; `canon_class: discovery` on next touch |
| (c) | Connective tissue | Illustration, funnel, strategy, instrument — serves (a)/(b), fails substitution only in noun | Kept in place; labeled (§2.2) |
| (d) | Skyzai/venture admixture | Truth/value depends on a named lane entity; belongs to 02_SKYZAI / 03_VENTURES | Moved to owning lane; K3 tombstone left (§2.1) |

## 2 · The move grammar

### 2.1 · Class (d) — relocate + tombstone (K3 archive-first)

1. **Move, byte-identical.** `git mv` (or plain move) the file/dir to the owning
   lane's dated intake: `02_SKYZAI/90_ARCHIVE/INBOUND_FROM_01_EMERGENTISM_2026_07_19/…`
   or `03_VENTURES/<owning-entity>/90_ARCHIVE/INBOUND_FROM_01_EMERGENTISM_2026_07_19/…`.
   **No body edits during a move** — content disputes are settled after arrival, in
   the owning lane, by that lane's route card.
2. **Tombstone in place.** At the origin path, leave `TOMBSTONE_<name>.md`
   containing: origin path, destination path, date, this envelope's receipt path,
   one-line verdict, mover (L4), K2 initial. Tombstones are permanent (K3: moved,
   never erased).
3. **Link repair is a separate row.** Inbound links from retained canon are listed in
   the envelope as their own rows; K2 initials each. No silent redirects.
4. **Nothing crosses a lane boundary except class (d).** Connective tissue stays where
   it is; pillars own their lanes (root AGENTS.md routing law).

### 2.2 · Class (c) — the label standard

- **Unsigned files:** frontmatter key `canon_class: connective` (vocabulary:
  `axiom | discovery | connective`) plus one marginal banner line at top of body:
  `> Canon class: connective (illustration | funnel | strategy | instrument) — not doctrine.`
  Named-instance nouns stay, wrapped in the illustration register ("for instance…").
- **K2-signed files (e.g., TRANSMISSION_STRATEGY):** the body is frozen — **never
  edit in-body.** The label rides in a sibling cover file
  `<name>.CANON_CLASS.md` carrying class, scope note, and the signed file's hash/date.
- **No churn campaign:** labels are applied to the rows K2 initials in this envelope;
  further files are labeled on touch, not by sweep-editing the tree.

### 2.3 · Classes (a)/(b) — untouched

No moves, no body edits, no renumbering. Optional `canon_class` frontmatter on next
natural touch only.
