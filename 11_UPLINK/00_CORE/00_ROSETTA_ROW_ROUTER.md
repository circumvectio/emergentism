## Read Law

1. Read **horizontally** inside a row when comparing like with like.
2. Read **vertically** across rows when translating doctrine into runtime, or
   runtime into public language.
3. Keep these three truth surfaces separate:
   - `02_SKYZAI/01_NOOSPHERE/P-SCORES.md` = code-only health
   - `02_SKYZAI/01_NOOSPHERE/ORGANISM_RUNTIME_TRUTH.md` = runtime/public claim boundary
   - dated organ receipts = bounded local proof

## Reorganization Law

- Prefer `README.md`, index, and router repairs before moving deep files.
- Move a file only when its **authority owner changes**, not just because its
  neighbors feel messy.
- In `01_EMERGENTISM/11_UPLINK/`, lane-home files are canonical; root duplicates are
  compatibility stubs unless explicitly reopened.
- In `02_SKYZAI/01_NOOSPHERE/07_PWAs/`, editable source beats generated mirrors and build output.
- Historical one-off tooling should be archived or guarded, not polished into
  fake live canon.

## Practical Tidy Rule

If a folder feels confusing, apply this order:

1. clarify the row owner
2. add or fix the local `README.md`
3. add missing cross-links to the next upstream and downstream rows
4. only then consider physical moves

That keeps the corpus legible without severing legacy links or smearing
authority across mirrors.
