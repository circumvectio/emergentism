## K2 Levels

### K2-0 — Absent / bypassed / invalid

The system has no meaningful sovereignty boundary, or bypasses it.

Examples:
- silent auto-execution
- fake signed flags
- internal bypass strings treated as approval
- unverifiable approval path

### K2-1 — Procedural envelope

The system has a meaningful approval boundary in process, but not yet a fully cryptographic one.

Typical features:
- K2 envelope exists
- execution requires a distinct approval step
- governance path is explicit
- but proof of signer identity may still be weak or partially stubbed

### K2-2 — Cryptographic authorization

The system requires a valid cryptographic signature for approval.

Typical features:
- canonical message construction
- cryptographic signature verification
- invalid signature hard-fails
- unauthorized signer hard-fails
- no pseudo-signing path may count as sovereign approval

### K2-3 — Embodied signer verification

The system verifies not only the cryptographic signature, but that the signer is the intended embodied natural person or authorized embodied constitutional operator.

Typical features:
- biometric or equivalent embodied verification
- cryptographic signature still required
- no confusion between device possession and personhood
