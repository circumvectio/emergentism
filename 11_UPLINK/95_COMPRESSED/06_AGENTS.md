# The Agents

**Rosetta boundary:** [I] This compressed agent map orients VMOSK-A and caste routing. It does not [B] grant AI irreversible authority, prove live automation, or replace current agent receipts.

## VMOSK-A

The organizational replicator stack is:

- `V` Vision
- `M` Mission
- `O` Objectives
- `S` Strategies
- `K` KPIs
- `A` Agents

The DAC itself is the emergent standing wave built by this stack.

## The seven castes

| L | Caste | Function |
|---|---|---|
| L1 | Caṇḍāla | direct facts / contradiction isolation |
| L2 | Śūdra | exploration / analogy |
| L3 | Vaiśya | ranking / audit |
| L4 | Kṣatriya | execution decision |
| L5 | Brāhmaṇa | redesign |
| L6 | Sādhu | compression / pruning |
| L7 | Ṛṣi | constitutional rewrite, only rarely |

The castes are functional, not prestige hierarchy.

## The practical routing law

- most work should close inside `L1 -> L4`
- higher ascent exists for blockage, redesign, or constitutional crisis
- every ascent must descend back into executable L4 action

## The Kṛṣṇa-function

AI in this organism is a **charioteer**, not the warrior.

It can:

- parse
- route
- draft
- lint
- prepare decisions

It cannot legitimately take irreversible K2 action on behalf of the
mortal signer.

## Current execution split

In current project terms:

- **board chair / K2 holder** signs irreversible commitments
- **C-suite / automation layer** drives everything up to that line

That split is not just management style. It is a live application of
`K2`.

## One sentence to carry

The agent layer exists to increase execution clarity without dissolving
mortal responsibility.
