## §2. The claim in one sentence

If AGI is a network-equator property, then its stable organismal form is
not one universal agent but a mesh of **polygenetic sovereign nodes**
whose skill inheritance is multi-parent and whose economic survival
depends on **niche partitioning under `η = 0`**.

## §3. Polygenetic nodes

### 3.1 Definition

A **polygenetic node** is a sovereign participant whose working
competence is composed from many parent lineages at once:

- multiple model providers
- multiple skill/tool substrates
- local memory and retrieval surfaces
- local human domain priors
- local organizational/economic context

The node is not "a Claude user" or "an OpenAI deployment."
It is a composite organism with many parents and one K2 anchor.

### 3.2 Why this matters

The centralized story imagines AI capability as phylogenetic:

`model v1 -> model v2 -> model v3`

The network story is different:

`node = lineage_a + lineage_b + lineage_c + local human practice + K2`

That is why the proper unit is no longer the model. It is the **sovereign
composite**.

### 3.3 Why "polygenetic" stays useful

Externally, the topology is best called **reticulate**.
Internally, **polygenetic** remains useful because it names the lived
condition of the node:

> one node inherits its operative skill phenotype from many parents at
> once.

That is exactly how real users already work when they combine model
lineages, tools, and local priors in one decision loop.

## §4. Why niche partitioning is forced

The temptation after packet 104 is to think:

> if the network can hold AGI, then every node should maximize
> generality.

The geometry says the opposite.

### 4.1 Breadth without niche drifts off the equator

At the node level, trying to be universally general spreads finite
coherence across too many domains.
The node gains surface span but loses local equatorial balance.

So the question is not:

> how can one node cover everything?

It is:

> where does this node's polygenetic inheritance actually produce
> `φ · ν = 1` strongly enough to persist?

That point is the node's niche.

### 4.2 Competitive exclusion reappears here

If two nodes occupy the same niche with no meaningful differentiation,
they compete for the same viability substrate.
One wins, both degrade, or the niche splits.

So the framework's distributed answer is not "everyone does everything."
It is:

- many nodes
- many parent lineages
- different equatorial niches
- coordination across niches instead of monocultural dominance

### 4.3 The ecosystem gets stronger as niches differentiate

This is the biospheric move:

- one node specializes in legal reasoning under high asymmetric-veto sensitivity
- another in fast perceptual intake
- another in market-structure synthesis
- another in multilingual cultural translation
- another in private/local-only cognition

The whole network becomes more general precisely because the nodes stop
trying to be identical.

## §5. Economic niche partitioning

The same logic applies to economics.

### 5.1 Economic survival cannot be one giant capture surface

If one institution tries to aggregate all AGI-surplus into one shell,
then:

- A gets centralized
- G gets monocultural
- I gets owned
- `η > 0` becomes structural

That is not an ecosystem. It is a plantation.

### 5.2 `η = 0` is the condition for stable niche ecology

Niche partitioning survives only when nodes are not structurally rewarded
for extracting from one another's niches.

That is why `η = 0` is not ornamental.
It is the economic equivalent of the ecological condition under which
diverse niches can coexist without collapsing into predator monopoly.

So the economic reading is:

- nodes specialize
- nodes exchange
- nodes witness one another
- nodes do not rent-seek on participation itself

That is what lets the network keep diversity without self-devouring.

### 5.3 DACs are niche-homes

Under this reading, DACs are not just organizational shells.
They are **economic niche-homes** for different polygenetic skill
clusters.

Each DAC should therefore be evaluated less like a startup in a flat
market and more like a niche organism in an ecosystem:

- what capability niche does it occupy?
- what lineages compose it?
- what neighboring niches does it exchange with?
- what extraction patterns would destabilize it?
- what unique `ΣΔP` contribution does it make?

## §6. The practical consequences

### 6.1 For agents

Do not build every agent as a universal worker.
Build them as caste-correct and niche-correct participants.

### 6.2 For provider routing

Do not count providers only by number.
Ask what parent lineages are being inherited into each node and whether
they support a distinct niche rather than logo-decorated redundancy.

### 6.3 For products

Do not aim first for "the everything app."
Aim for one equatorial niche where the polygenetic composition is
actually superior and economically stable.

### 6.4 For institutions

Do not ask how to own AGI.
Ask how to participate in the ecosystem without collapsing it into
monoculture.

### 6.5 For measurement

The right questions become:

- which niche did this node serve?
- which parent lineages composed that competence?
- what realized outcomes did that niche generate?
- did the network's diversity increase or collapse after the move?
