## 1. The reframe

### 1.1 Sovereign statement (2026-04-25)

> *"Each dac has an Uplink and Downlink which is like inbound and outbound logistics in partner value chain but for DACs. So Uplink < Operations < Downlink < ...."*

### 1.2 What was missing

Packet 177 §5.5 specified that a DAC inherits a 12-folder organism kit (Factory · Wallet · Equity · Credit · DEX · POS · Agents · Cortex · Nexus · AIA · Three-Stage Process · Covenant). That spec covered **the inside of the cell**. It did not name the **boundary** through which the cell exchanges with adjacent cells.

Without a named membrane:
- Federation between DACs is unspecified
- Inter-DAC transactions have no canonical surface
- Boundary-attaching concerns (privacy, dispute, customs/jurisdictional interface, commons accounting) lack a structural home
- OQ-MENEXUS-2 from packet 184 (federation protocol: thin/medium/thick) cannot be answered

### 1.3 The membrane named

**Uplink** = the DAC's inbound surface — what arrives from counterparty DACs (procurement, deliveries, communications, data, value transfers received).

**Downlink** = the DAC's outbound surface — what is shipped to counterparty DACs (deliverables, communications, data published, value transfers sent).

Both operate under the DAC's K2 authority. Both are receipts-witnessed via OFN. Both compose existing substrate.

## 2. As-Above-So-Below — the fractal

### 2.1 Internal cognitive flow (Three-Stage Process)

Inside one DAC, cognition flows:
**IS → COULD → SHOULD → ACT** (Three-Stage Process)

This is the four-organ separation per packet 177 §5.5 and `01_EMERGENTISM/00_SEVENFOLD_FOUNDATION_ROOT.md`.

### 2.2 Inter-DAC logistics flow (Membrane)

Between DACs, logistics flows:
**Uplink → Operations → Downlink**

Operations itself contains a full Three-Stage Process. So the chain nests:

```
[DAC-A Uplink] → [DAC-A Operations = Three-Stage Process] → [DAC-A Downlink]
  → [DAC-B Uplink] → [DAC-B Operations = Three-Stage Process] → [DAC-B Downlink]
  → [DAC-C Uplink] → [DAC-C Operations = Three-Stage Process] → [DAC-C Downlink] → ...
```

Each cell has the same internal pattern; cells chain laterally via membranes; the organism emerges from the federation of cells through their boundaries.

### 2.3 Why As-Above-So-Below holds

The same three-phase pattern recurs at multiple scopes:
- **Within one organ** (e.g., TheCircle): observation → inference → output
- **Within one DAC**: IS → COULD → SHOULD → ACT
- **Between DACs**: Uplink → Operations → Downlink
- **Within the organism**: cell-cluster → cell-cluster → cell-cluster

Fractal recurrence is the signature of a coherent core state, not coincidence. The membrane primitive is what makes the recursion legible at the inter-DAC layer.

## 6. Name-collision flag

`01_EMERGENTISM/11_UPLINK/` is the framework-level orientation-packet folder (where these packets live). DAC-level Uplink is the inbound logistics surface for a specific DAC.

**Same word-function** (telecom sense: signal arriving at the receiver), **different scope** (framework receives orientation; DAC receives logistics).

Tolerable with scope-prefix convention, but worth resolving before specs are written under either word. If the collision becomes friction, candidate alternatives:
- `Inbound` / `Outbound` (Porter-clean; loses the telecom market fit)
- `Intake` / `Ship` (verb-clean; loses the symmetry)

**Charioteer lean:** keep `Uplink` / `Downlink` at DAC scope; document the dual usage; address only if friction surfaces. **OQ-MEMBRANE-1** flagged for sovereign disposition.
