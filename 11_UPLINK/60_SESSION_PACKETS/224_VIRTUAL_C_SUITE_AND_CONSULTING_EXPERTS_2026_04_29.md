## 1. The Decision

**Two interlocking organizational structures**, both built on the dual-model substrate (packet 216), both governed by K2/PRISM:

1. **The Virtual C-Suite** — seven permanent roles mapped to the L1-L7 Rosetta castes. The C-suite IS the polygenic tree at organizational scale. 2. **The Consulting Experts** — twelve on-demand domain specialists, each a queryable body of expertise. Reached via WHISPER + paid via API PAY. Engaged when the C-Suite encounters a question outside its standing competence.

This packet *perfects the blueprint* by giving the implementation lane (sprints S1-S8 from packet 223) a clear organizational frame: who does what, when, and through which substrate primitive.

## 3. Consulting Experts — Twelve On-Demand Specialists

The C-Suite handles standing executive function. Some questions exceed the C-Suite's competence and benefit from domain specialists. The framework engages **twelve consulting experts**, each:

- A **queryable body of expertise** (a fine-tuned LeWorldModel sub-domain, a real human consultant, or a curated corpus + LLM ensemble — implementation choice per engagement)
- Reached via **WHISPER** (intent-authored DM)
- Paid via **API PAY** (per-engagement, η=0 economy)
- Subject to **K2 acceptance** before any of their advice binds the DAC
- Recorded as **FLOW receipts** (durable; the engagement is auditable)

| **E4** | **LLM / JEPA / World Models** | LeWorldModel architecture choices (per Q4 reconciliation); fine-tuning strategies; substrate evolution | S1 substrate spine; model swaps; performance regressions |

**Engagement workflow:**

1. **CSuite role identifies the gap** — the standing C-role recognizes "this question exceeds my competence; we need an expert."
2. **APU proactivity surfaces the engagement proposal** — Five-Ws filled (When = engagement window; Where = remote / async; Who = expert E_n; What = the specific question; Why = the parent Mission)
3. **WHISPER transmits the engagement** — `@expert_E_n + intent` → AI drafts → K2 accepts → wrap → expert
4. **Expert responds** — through their preferred surface (WHISPER reply, RELAY broadcast for public-facing, FLOW signed deliverable)
5. **C-Suite integrates** — CAO ranks the expert's input against existing constitution; CEO binds (or refuses) with K2 signature
6. **Receipt persists** — FLOW receipt of the engagement; both the question and the expert's deliverable are part of the audit trail

**Why twelve, not more:** the cardinality is heuristic. The number can grow as the framework encounters new domains; the structure (queryable, WHISPER-reached, API-PAY-paid, K2-bound) is the invariant. Twelve is a minimum-viable starting set covering the framework's current substrate-implementation lane.

## 7. What Packet 224 Does NOT Claim

1. **It does not claim the C-Suite or experts are currently instantiated.** Per the operational picture above, this is a *target architecture* with a phased implementation path. Phase 0 (prompted personae) is achievable in S1; Phase 2 (cross-DAC niche-graph integration) requires S4 to land first.
2. **It does not lock the twelve consulting expert domains.** The list is a minimum-viable starting set covering the substrate-implementation lane. New experts can be admitted under the same engagement workflow; the structure (queryable, WHISPER-reached, API-PAY-paid, K2-bound) is the invariant, not the names.
3. **It does not change the K2 boundary.** Every binding act still requires K2 (private DAC) or PRISM-governance (public DAC). The C-Suite proposes; K2 binds. Experts advise; K2 accepts or refuses. This is consistent with packets 207, 213, 215, 216, 222.
4. **It does not replace the four organs.** The Three-Stage Process organs (TheCircle, RealityFutures, APU, Evolutionary.Network, Skyzai) operate at framework scale across DACs. The C-Suite operates at DAC scale within each DAC. Both are simultaneously live.
5. **It does not remove the Krishna-function discipline.** The Krishna-function (06_AGENTS) — recognize / name / dispatch / return through L4 — operates *inside* each C-role's reasoning. The CEO's binding decision is itself a Krishna-function-dispatched action. The C-Suite is structured *by* the Krishna-function, not in competition with it.
