## 0. Verdict

Uplink is no longer only a compressed readout of the underlying ventures.

It is now the **bidirectional model layer**:

1. underlying ventures emit source truth, receipts, blockers, and claims upward;
2. Uplink compresses those signals into organism-level models;
3. the models emit bounded obligations back down into the source owners;
4. downstream work returns receipts, state deltas, or refusals;
5. Uplink recompiles only after those artifacts exist.

This is what "symbiotic" means in this corpus: no project is allowed to become a silo, and no cross-project connection is allowed to become an overclaim.

---

## 1. The Symbiosis Law

Every live venture must answer two questions.

**What does this surface give the organism?**

Examples: receipts, demand, source truth, identity graph, capital access, deal pressure, public signal, governance memory, tooling, runtime proof.

**What does this surface need from the organism?**

Examples: claim discipline, K2 boundary, proof grammar, source routing, execution infrastructure, receipt schema, audience, capital, legal membrane, cross-organ intelligence.

A surface is symbiotic only when both answers are explicit and artifact-bound.

If the answer is merely "brand synergy", "strategic alignment", "AI layer", "tokenization layer", or "future integration", the claim is not yet symbiotic. It is target-state narrative.

---

## 2. Compiler Loop

The Uplink compiler loop is:

```text
source venture state
  -> Uplink extraction
  -> claim-tier normalization
  -> cross-surface model
  -> downstream obligation
  -> source-owner work
  -> receipt / refusal / state delta
  -> Uplink recompile
```

The important inversion is that Uplink may not simply summarize a venture. It must also say what the venture owes next if the summary is to become truer.

Compression without downstream obligation creates theater.
Downstream work without Uplink recompile creates drift.

---

## 3. The Four Outputs Uplink May Emit

Uplink can lawfully emit four kinds of downstream work.

| Output | Meaning | Required return artifact |
|---|---|---|
| **Receipt ask** | "Prove this loop or stop saying it." | receipt, trace, signed artifact, deployment evidence, test result |
| **Boundary ask** | "Separate these claims before they counterfeit each other." | taxonomy update, claim badge, external wording rule, source route |
| **Interface ask** | "Define how two surfaces may touch without merging." | interface object, data contract, API boundary, legal/counsel note |
| **Model ask** | "Refine the theory because the work exposed a contradiction." | audit packet, falsifier, revised model, source correction |

Anything else is advisory and should not be treated as an execution command.

## 5. Non-Merge Principle

Symbiosis is not merger.

The organism becomes stronger when boundaries are clear enough for surfaces to cooperate without stealing each other's evidence.

Therefore:

- QNTM (the institutional MPC/ZK-Identity rail) must not borrow Agentz's tokenization traction.
- Agentz must not borrow QNTM (the institutional MPC/ZK-Identity rail)'s banking ambition.
- Circle must not borrow APU's judgment authority.
- APU must not borrow Skyzai's execution authority.
- SPECTRE must not borrow live settlement truth before the rooted commercial loop earns it.
- Foundation doctrine must not borrow runtime truth where the code or receipts do not yet exist.
- AIA must not borrow the whole immune-system role; it is the Automated Information Architect, while SPECTRE remains the routing/immune design lane and Cortex remains memory/witness.

The correct verb is **interface**, not merge.

## 6. Model Refinements From This Pass

### 6.1 Uplink is a compiler, not a brochure

The old risk was stale compression. The new rule is: every Uplink synthesis should include either a source pointer, a downstream obligation, or a reason it remains interpretive only.

### 6.2 Products are proof engines

Circle, Pay, Events, Academy, QNTM (the institutional MPC/ZK-Identity rail), and Agentz are not separate stories. Each should become a proof engine for at least one other surface:

- Circle gives public observation and trust surface to the organism.
- Pay / Events give receipts and revenue proof to MENEXUS / Skyzai.
- Academy gives relationship artifacts to AGENTZ.
- QNTM (the institutional MPC/ZK-Identity rail) gives a future money-side interface if and only if lawfully earned.
- Agentz gives deal pressure and tokenization proof demands back to the organism.

### 6.3 Claims must carry return paths

A claim is healthier when it names what artifact could make it stronger or kill it.

Examples:

- "relationship moat" -> artifact ledger, cohorts, named mandates, SSB interactions;
- "merchant proof" -> receipt / settlement / statement pair;
- "Circle trust surface" -> public failure log and provider-basin diversity;
- "QNTM (the institutional MPC/ZK-Identity rail) interface" -> status-only membrane object, not shared authority;
- "SPECTRE route energy" -> replay corpus, held-out validation, live-path wiring.

### 6.4 The next work should descend by obligation, not by folder

Do not ask "which project should we polish?"

Ask:

1. Which claim currently creates the most downstream confusion?
2. Which receipt would make the largest number of other claims less rhetorical?
3. Which interface is needed so two surfaces can cooperate without merging?
4. Which stale model should be killed before it generates work?

### 6.5 Boundary model governs compression

Packet 130 is now the boundary audit for organism summaries. Any summary that
touches VMOSK-A, Cortex, AIA, SPECTRE, P-scores, or runtime truth must preserve
these separations:

- VMOSK-A = replicator stack;
- Cortex = memory / witness / drift surfacing;
- AIA = Automated Information Architect / gardener metabolism;
- SPECTRE = later routing / selection / immune design lane, with prototype-real
  BEAM route energy but no live network proof;
- P-scores = internal health signals, not core state proof or public launch
  authorization.
