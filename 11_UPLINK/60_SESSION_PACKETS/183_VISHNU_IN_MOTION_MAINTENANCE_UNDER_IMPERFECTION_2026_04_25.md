## 0. Axiomatic guard

This packet does **not** specify twelve protocols.

It names one layer:

> **Viṣṇu-in-motion** — the discipline by which the organism stays honest while reality perturbs it off the equator.

Birth has a grammar. Death now has a grammar. This packet names the missing middle: not static preservation-at-rest, but operational preservation while the world bends, delays, confuses, exhausts, disputes, translates, and partially breaks the organism.

---

## 1. The shape

Birth is **Brahmā** (○).

Death / dissolution is **Śiva** (•).

Preservation-at-rest is **Viṣṇu** (⊙).

The missing layer is **Viṣṇu-in-motion**: the operational form of preservation when perturbation is continuous.

The organism has been strongest in peak-state doctrine:
- clean genesis
- clean covenant
- clean inheritance
- clean revocation
- clean death

Ordinary life is not clean. Ordinary life is perturbation. Perturbation is most of what a living organism does.

Therefore this is not a new architecture. It is the maintenance doctrine for the architecture already named.

---

## 2. Why not twelve packets

The twelve observed gaps cohere under one question:

> What does the organism do when reality does not cooperate?

If each gap is packeted independently, the framework scatters:
- mistake becomes accounting-only
- dispute becomes legal-only
- privacy becomes ZK-only
- capacity becomes medicalized
- translation becomes marketing
- practice becomes procedure

That would be Brahmā overstep: generating scaffolds because generation feels productive.

The opposite error is Sādhu overcut: name the layer, refuse to specify anything, and let drift continue.

This packet takes the middle way:
1. Name the layer.
2. Audit the facets.
3. Decline premature specification.
4. Ask sovereign which thread, if any, should be pulled first.

## 4. Audit against current CANON

### 4.1 Fully absent

- Mistake / rollback
- Dispute
- Capacity / fatigue
- Reputation / forgiveness
- Translation fidelity

These have no stable grammar yet. They should not be assumed solved by Cortex, AIA, or covenant language.

### 4.2 Partially addressed but not named as doctrine

- Privacy / selective disclosure
- Time / heartbeat
- Upgrade / migration
- External law interface
- Agent liability
- Commons accounting

These have fragments in runtime, Cortex, AIA, covenant, SoResFi, or public-surface language. The fragments are real, but they do not yet compose into one maintenance doctrine.

### 4.3 Cannot be replaced by doctrine

- Practice / formation

Practice can be named, scaffolded, and protected, but it cannot be packeted into existence. If the organism does not sit, pause, confess uncertainty, and return to the axiomatic guard, no document will do that work for it.

## 5. Charioteer ordering lean

If sovereign chooses to pull a thread, the highest-leverage order is:

1. **Privacy / selective disclosure**
   Financial services make this load-bearing before scale.

2. **Mistake / rollback + dispute**
   These should be paired: honest error and honest disagreement are sibling states.

3. **Capacity / fatigue**
   This protects K2 from silently degrading while still appearing sovereign.

4. **Upgrade / migration**
   This enables live evolution without breaking continuity.

5. **Translation fidelity**
   This prevents public, investor, legal, and agent surfaces from drifting apart.

Practice stays named but unspecifiable. The right move there is formation, not another packet.
