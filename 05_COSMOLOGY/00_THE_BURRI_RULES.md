## 0. Type law: distinguish actual carrier from possible content

This distinction is mandatory and is **not a seventh rule**:

```text
TokenModality := actual | merely_possible
TopologyMode := structural | actual | possible

D0…D3,D6 abstract register/crossing nodes = structural
D4 tokens/events = actual
D5 contents = merely_possible (`possible` in topology export)
```

- `D4` contains causal actuality: embodied means, present model tokens,
  performed ranking/selection events, attempted/performed action, factual
  record, and receipts.
- `D5` contains counterfactual content: alternatives, future referents,
  rankings-as-represented, and candidates for selection.

A D4 actual token can represent D5 merely-possible content. Topology exports
retain `possible` as the short schema value for `merely_possible`; they must
not type a present model or selector state as possible.

`D4→μ₄→D5` asks whether counterfactual agency emerges. Commitment uses a
selected D5 option and D4 means to produce an attempted D4 action. The world,
not the selector, determines the resulting history.

Solid edges denote enacted D4 relations. Dotted edges denote modeled D5
alternatives. Curved return edges denote feedback. Evidence must remain legible
without color.

## BR-3 · Constraint

> Higher-level organization may restrict or reweight lower-law-admissible
> trajectories; it may not manufacture a forbidden one.

For a lower-level transition kernel `K_X` and higher-level constraint `C`:

\[
\operatorname{support}(K_X^C)
\subseteq
\operatorname{support}(K_X).
\]

This is the operational form of downward constraint. A membrane changes which
chemical trajectories occur; a plan changes which physically possible action
is attempted; a law changes incentives and permissions. None suspends the
underlying admissibility conditions.

Constraint claims must state:

- the lower-level possibility set;
- the higher-level variable;
- the reweighting or restriction mechanism;
- an intervention on the constraint;
- the expected lower-level distribution change;
- a null model and kill criterion.

**Kill criterion:** if the higher-level variable adds no intervention-sensitive
predictive value, remove it or treat it as a summary. If it requires a forbidden
trajectory, the model is false.

## BR-4 · Commitment

> A finite agent combines a fallible D5 option field with D4 means to attempt
> one D4 action; accountable authorization determines whether that attempt is
> normatively valid, not whether it is causally representable.

\[
\chi_t:(X_t,\Omega_t,M_t,V_t,U_t,G_t)
\longrightarrow(a_t,q_t),
\qquad a_t\in Action\cup\{\bot\}.
\]

`U_t` contains an `AuthorizationAssessment`. A potentially valid envelope
contains:

```text
principal + mandate + scope + consent + custody + expiry/revocation
+ contest path + actor + consequence bearer set
```

The assessment is a validated tagged union. `valid` requires both a complete
envelope and evaluated principal authority, mandate/scope, actor match, current
consent, custody, non-expiry/non-revocation, contest availability, and complete
bearer coverage for the attempted action. A complete but failed envelope is
still `invalid`. `invalid` requires a supplied defective record plus reasons;
`absent` requires a null envelope plus reasons; `not_required` is limited to an
explicitly nonconsequential scope. Commitment status is derived from the union,
not copied from an unvalidated label.

If no action is attempted because it is refused or physically unavailable,
`a_t=⊥`; no action transition is submitted. Invalid or absent authorization is
recorded separately. A governed channel `χ_t^J` refuses it, but if an actor
nevertheless attempts the action, the descriptive causal kernel receives it.

For an attempted action, `q_t:CommitmentReceipt` records selection, actor,
physical availability, authorization status and nullable envelope, means,
intention, every affected bearer, payers, beneficiaries, and expected result.
The environment separately returns:

\[
(X_{t+1},r_{t+1})
\sim K_t(\cdot\mid X_t,a_t,E_t).
\]

`r_{t+1}:OutcomeReceipt` records observed consequence. It is nullable while
the loop waits for, or never receives, a world observation. Let `E_q` be the
bearer set anticipated by the commitment and `R_r` the full bearer set found
by observation. A valid receipt requires `E_q⊆R_r`, not equality. Every bearer
in `R_r` has at least one observation, and observations are unique by
`observationId` or `(bearerId,measureId)`, so one bearer may lawfully carry
several distinct measures. The drift `R_r\E_q` remains valid evidence of an
unforeseen consequence; it simultaneously fails retrospective Justice and the
prior authorization's bearer coverage. Evidence is not discarded merely
because it indicts the commitment.

If `a_t=⊥`, a non-null world receipt must have
`receiptCause=ambient_observation` and null action identifiers; it is not an
outcome caused by `⊥`. The selector cannot manufacture its own outcome. `G_t`
is an input to the selector, so updating `G_{t+1}` can change the next action
distribution.

The existing **G7 vocabulary** remains available inside BR-4 as the selected
typed union `G7=M4⊔F3`. Only `M4` contains move labels; `F3` contains boundary
frames, not actions. The Burri Rules type the conditions under which an option
is modeled and an actual move is selected, authorized, attempted, and
receipted. G7 is not an exhaustive ontology and does not prohibit an eighth
useful description.

**Kill criterion:** if intention, action, authorization, and consequence cannot
be distinguished, the commitment is not auditable and fails closed.

## BR-5 · Reflexive Return

> Every commitment receipt enters the return, and any later outcome receipt
> joins it. Either can update both the agent's model and the selector that uses
> it.

\[
(M_{t+1},G_{t+1})
=\operatorname{Loop}(M_t,G_t,q_t,r_{t+1}).
\]

Here `r_{t+1}\in OutcomeReceipt\cup\{null\}`. The commitment receipt is not a
decorative prelude to the world receipt: it records execution-relevant facts
such as refusal, physical unavailability, authorization status, and expected
bearer deltas. New information in `q_t` may therefore change `M` or `G` while
`r_{t+1}=null`. A null update is justified only when the available `q_t` is
genuinely non-informative for the current state and `r_{t+1}=null`.

| Return case | Required observable contract |
|---|---|
| informative `q_t`, `r_{t+1}=null` | at least one of `M` or `G` changes in the canonical witness case |
| non-informative `q_t`, `r_{t+1}=null` | `M` and `G` remain unchanged; this is the genuine null update |
| non-null `r_{t+1}` | the world receipt is evaluated as a further independent input |

The return preserves three gaps:

| Gap | Difference |
|---|---|
| cognitive | observed territory versus model prediction in a common observation space |
| execution | intended versus performed commitment |
| outcome | expected versus observed consequence |

Negative feedback means a declared error decreases. Positive feedback means
divergence or self-confirmation increases. These signs are dynamical, not
moral.
