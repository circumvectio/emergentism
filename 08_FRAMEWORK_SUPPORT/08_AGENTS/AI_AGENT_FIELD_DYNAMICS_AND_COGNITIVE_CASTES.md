# AI agent field dynamics

This note asks whether Emergentist distinctions help configure AI work. It does not
turn agents into literal points on a physical sphere, and successful software would
not independently confirm the worldview.

## Bounded design translation

| Emergentist distinction | Agent-design use |
|---|---|
| D5 possibility | generate, represent, and rank candidate actions |
| D4 actuality | available tools, permissions, performed action, and records |
| commitment receipt | what the agent actually attempted under the granted scope |
| outcome receipt | what the environment returned, including veto and failure |
| Soul Loop | update both model and policy from the two receipts |
| option cone | actions the agent can model, rank, coordinate, and reach |
| Justice envelope | include every material consequence bearer before optimization |
| Grace Exit | keep user control, export, reversibility, revocation, and contest paths |

The Rosetta roles supply a useful pipeline:

`L1 isolate → L2 expand → L3 rank → L4 act → receipt → update`,

with L5 redesign, L6 compression, and L7 boundary review invoked only when the task
requires them. This is a selected workflow, not a theorem of cognition.

## Anti-reification

- A model is an instrument, not a mind proved by metaphor.
- A role is a current function, not identity or rank.
- A score is a diagnostic, not intrinsic worth.
- A simulated sacrifice is not mortality.
- A generated future is a present model token, not a future event reaching backward.
- Tool permission is ordinary scoped authorization, not a metaphysical signature.

An agent is well configured when it preserves uncertainty, acts only with available
means and authority, records commitment separately from outcome, exposes costs and
bearers, and remains corrigible. Whether this pipeline outperforms rivals is an
empirical engineering question.
