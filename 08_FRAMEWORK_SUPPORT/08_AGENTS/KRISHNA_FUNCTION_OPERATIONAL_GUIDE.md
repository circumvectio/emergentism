## 4. DRAFT — Action Proposal

**Purpose:** Propose actions that raise φ (coherence) and/or ν (viability), with measurement proxies.

### 4.1 Action Design Rules

1. **Raise φ or ν, one axis at a time.** Under the φ·ν = 1 model, the operational move targets one measured axis while preserving the other as an explicit constraint.
2. **Smallest defensible change.** Not maximum optimal. The minimum action that moves the needle.
3. **Measurable proxy.** Every action must have a metric that indicates success within the session's time horizon.

### 4.2 Action Template

```yaml
draft_action:
  description: <what>
  owner_caste: <L1-L7>
  phi_effect: <raise|hold|lower>  # coherence impact
  nu_effect: <raise|hold|lower>   # viability impact
  measurement_proxy: <metric>
  time_horizon: <in_time|above_time|against_time>
  risk: <what could go wrong>
  rollback: <how to undo>
```

### 4.3 DRAFT Checklist

- [ ] Does this action raise φ or ν?
- [ ] Is it the smallest defensible change?
- [ ] Does it have a measurement proxy?
- [ ] Is the owner caste correctly assigned?
- [ ] Is the rollback path clear?
- [ ] Does it respect all boundaries (K2, K4, η=0)?
