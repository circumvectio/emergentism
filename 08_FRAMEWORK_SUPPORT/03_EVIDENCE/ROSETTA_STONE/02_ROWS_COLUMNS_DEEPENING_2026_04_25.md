# Rosetta Rows and Columns: Deepening Note

**Date:** 2026-04-25
**Status:** Working doctrine / audit scaffold
**Depends on:** [README.md](README.md), [00_THE_MASTER_ROSETTA.md](00_THE_MASTER_ROSETTA.md), [D_SERIES_ROWS/00_GENERATIVE_TABLE.md](D_SERIES_ROWS/00_GENERATIVE_TABLE.md), [00_SELECTION_METHODOLOGY.md](00_SELECTION_METHODOLOGY.md)
**Purpose:** Clarify how to read rows and columns without turning the Rosetta Stone into either proof-by-aesthetic-fit or a flat synonym table.

---

## Core Distinction

The Rosetta Stone is a grid, but the grid is not the thing.

- A **row** is one L-level read horizontally across domains.
- A **column** is one domain's projection of the same operator geometry across levels.
- A **cell** is a claim that one substrate expresses one operator-position in a specific way.

The row tests coherence. The column tests continuity. The cell carries the evidence tier.

This means no column should be treated as equally strong simply because it sits in the table. The operator and geometry columns are anchors. Psychology, politics, mythology, computation, neuroscience, linguistics, and initiatory traditions are projections with different evidence weights and different failure modes.

---

## The Row Law

A row earns its place when it can be read as a single functional sentence across domains.

For each L-level, ask:

1. Does the row name the same problem?
2. Does it show the same operator motion?
3. Does it have the same pathology when distorted?
4. Does it transform into the adjacent rows without a category jump?
5. Does it mirror its opposite row around L4?
6. Does the evidence come from the source domain rather than from the framework's wish to fit it?

If the row does not survive these questions, the row is not false automatically. It is downgraded: from established or structural to interpretive or conjectural.

### Row Symmetry

The most important row test is not simple order. It is mirror symmetry around L4.

| Pair | Symmetry Question |
|---|---|
| L1 / L7 | Does raw boundary pressure mirror transcendent boundary closure? |
| L2 / L6 | Does first negation / immune refusal mirror axiomatic refusal? |
| L3 / L5 | Does productive system-building mirror contemplative system-holding? |
| L4 | Does the center hold phi = nu = 1 without collapsing into either side? |

[S] The center is not a compromise. It is the maximum-balance point. On the corrected geometry, potential as φ · ν is always 1 on S²; what varies is balance, B = sin theta. Any row reading that treats L5-L7 as simply "higher rank" has drifted from the source geometry.

## The Column Law

A column earns its place when the same domain preserves the row order without coercion.

For each column, ask:

1. Is this domain naturally levelled, staged, stratified, or cyclic?
2. Are its source categories independent of the framework?
3. Does the domain preserve adjacency, not only isolated matching?
4. Does it preserve the L4 center as a balance or commitment node?
5. Does it preserve at least one mirror pair?
6. Does it produce a diagnostic, prediction, or falsifiable contrast?
7. Are absences and failed fits recorded?

A column with one brilliant correspondence and six stretched cells is not a strong column. It may still be valuable as a poetic or exploratory surface, but it should not be used as evidence for convergence.

### Column Classes

| Class | Role | Standard |
|---|---|---|
| Anchor columns | Operator, geometry, equation, balance | Define the formal generator; must be internally consistent. |
| Structural columns | Varna, reasoning, pramana, -ology, polity | Test whether social and epistemic functions cohere with the generator. |
| Domain columns | Psychology, computation, mythology, neuroscience, politics, linguistics | Test whether independent knowledge domains preserve the row pattern. |
| Translation columns | Public language, pedagogy, cognitive-role routing, institutional operations | Make the structure usable without pretending usability is proof. |
| Speculative columns | Weak, newly proposed, or highly interpretive mappings | Must carry explicit tier, failure condition, and exclusion notes. |

The master table can contain all five classes, but readers must not confuse them. The strongest table is not the largest table. It is the one where every column knows what kind of claim it is making.

## The Cell Law

The real Rosetta object is the cell:

```text
operator position x domain substrate -> expression, tier, failure mode
```

A mature Rosetta cell should carry:

- **L-level:** the row position.
- **Operator:** the geometric or functional motion.
- **Domain expression:** the local name inside the source domain.
- **Evidence tier:** established, structural, interpretive, or conjectural.
- **Source basis:** text, dataset, model, artifact, practice, or observation.
- **Fit reason:** why this is the right row, not merely a similar word.
- **Adjacent check:** why the previous and next rows still make sense.
- **Mirror check:** how the opposite row reflects or inverts it.
- **Kill criterion:** what would make the mapping fail.

Without this, the table is easy to admire and hard to audit. With it, the table becomes a research instrument.

## Deep Reading

Read horizontally first:

```text
L-level -> operator -> functional problem -> domain expressions
```

Then read vertically:

```text
domain -> L1 -> L2 -> L3 -> L4 -> L5 -> L6 -> L7
```

Then read diagonally:

```text
adjacent transitions + mirror pairs + L4 center
```

The horizontal read asks: "Is this one row?"

The vertical read asks: "Is this one column?"

The diagonal read asks: "Is this one geometry?"

If all three hold, the mapping strengthens. If only one holds, it is probably analogy. If none hold, archive it or mark it as a failed fit.

## Guardrails

1. The Rosetta Stone is calibration data, not proof.
2. Seven is a chosen resolution of the model, not a discovered constant.
3. Piaget and Kohlberg are not independent in the strict sense; Kohlberg builds on Piaget.
4. High-level developmental and contemplative mappings carry WEIRD and selection bias risk.
5. P/B drift must be corrected: on S², P∞ = φ · ν = 1; balance B varies.
6. "Every column" is an aspiration and rhetorical shorthand; audit work must specify which columns actually hold.
7. Failed mappings are not enemies of the framework. They are how the instrument gets calibrated.
