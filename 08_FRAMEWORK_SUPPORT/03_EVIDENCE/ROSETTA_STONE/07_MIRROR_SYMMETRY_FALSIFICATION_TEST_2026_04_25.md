## 0. Axiomatic guard

The current framework treats the L1↔L7, L2↔L6, and L3↔L5 mirror assignments
as an `[I]` design hypothesis. Antipodal/reflection operations on `S²` are
mathematical, but the selected seven samples and their domain meanings are not
forced, and the two poles remain distinct. The return is `r₆:D6↝D0`.

This means: **for any column claiming to be a 7-projection of S², the L1↔L7, L2↔L6, L3↔L5 pairs must show *functional inversion* — same kind, opposite direction.** If a column passes this test cleanly, it's a genuine S² projection. If it fails, it's a forced fit or a linear domain pasted onto a circular geometry.

This document operationalizes that test. **Stronger than evidence-tier annotation alone**: instead of marking a column "[S]" or "[S]," we score each MIRROR PAIR within each column. A column with 3 strong mirrors is doing real geometric work. A column with 3 fail-grade mirrors is decorative.

## 1. The protocol

For each 7-row column in the Rosetta, score the three mirror pairs:

- **L1 ↔ L7:** Boundary mirror (near-pole positions). The pair should show *same kind of boundary pressure, inverse direction*. L1 = pre-social raw threat (boundary from below); L7 = post-social transcendent closure (boundary from above).
- **L2 ↔ L6:** Half-balance mirror (B = 0.5). The pair should show *same balance position, inverse mode*. L2 = first-contract / gathering-in; L6 = renunciation / refusing-out.
- **L3 ↔ L5:** High-balance mirror (B = √3/2). The pair should show *same high-balance position, inverse modality*. L3 = productive-active building; L5 = contemplative-receptive holding.

**Scoring:**

| Score | Meaning |
|---|---|
| ✓ STRONG | Functional inversion is immediate and obvious without framework glasses. A naive reader would notice the symmetry. |
| ◐ PARTIAL | Mirror holds with framework reading; not obvious to an outsider; defensible but interpretive. |
| ✗ FAIL | No functional mirror. The L5/L6/L7 entries seem linearly higher than L3/L2/L1, not reflectively paired. |

**Per-column FALSIFICATION SCORE:** count the failures across the 3 pairs.
- 0 fails (all STRONG/PARTIAL): column is a genuine S² projection
- 1 fail: column is mostly clean but has a structural weak point
- 2 fails: column is partially forced
- 3 fails: column is linear, not reflective; treat as decorative

Tyrant rules by force from below the social contract; Theocrat rules by high-priority mandate from above the social contract. Same single-source structure, inverse claim of legitimacy (force vs mission-critical). - **L2 Democracy ↔ L6 Anarchy:** ✗ FAIL. Plato's *Republic* explicitly treats democracy as DEGENERATING INTO anarchy as a single phase. Aristotle treats these as the same structural regime distinguished by motive (good aristocracy vs corrupted oligarchy); Plato distinguishes them by metal (silver vs gold). The mirror holds at the structural level (rule-by-elite) but the inversion direction is more "moral upgrade" than "geometric reflection."

**Falsification score:** 1 fail (Democracy↔Anarchy), 2 partial. **Plato's Regimes column is *partially forced*.** It survives the test but the forcing at L2↔L6 is real.

This matches the sovereign's Pressure Point #3 observation: Plato's center is Aristocracy, not Timocracy. The framework inverts Plato. Mirror symmetry test confirms: the inversion is real and partially forced.

## 4. What this test buys the framework

### 4.1 A real falsifiability scoreboard

The framework can now publish, for every column, a per-mirror-pair score. This makes the kill criterion concrete: a column with 3 fails should be either:
- **Demoted from canonical to interpretive** (the mirror geometry doesn't apply)
- **Removed from the Master Rosetta** (it's pattern-matching, not S² projection)

### 4.2 A defense against the column-taxonomy critique

Pressure Point #1 says "15 columns demand explanation, but 7 are derived." Mirror symmetry test cuts orthogonal to this: a derived column can still pass or fail the mirror test. Derivation doesn't pre-determine score; geometric structure does.

The Pramāṇa column is internally generated (Vedāntic, traditional), but it scores 3-strong. The Computation column is also internally generated (CS-traditional), but it scores 2-fail. The TEST is independent of derivation — it's about whether the column actually projects S² geometry or just mimics it.

The 5-column compression the sovereign recommends in Pressure Point #1 — Psychology L1-L4, Plato, Initiatory, PIE, Neuroscience L1-L4 — could be replaced by a **mirror-test-passing compression**: keep the columns that score 0-fail, demote the others.

### 4.4 A framework-improvement loop

When a column fails the mirror test, the test itself suggests two paths:
1. **Accept the column as decorative.** Mark it [C] conjectural; remove from canonical V-bis.
2. **Re-derive the L5/L6/L7 entries from the mirror principle.** Force the framework to PRODUCE the missing inverse for each L1/L2/L3 entry.

If those predictions match the source domain, the framework wins. If not, the column is decorative.
