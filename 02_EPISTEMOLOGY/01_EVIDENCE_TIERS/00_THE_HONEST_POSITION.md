## The Antifragility Rule

This document is incomplete unless a serious reader can answer four questions about a major claim:

1. What tier is it currently in?
2. What would upgrade it?
3. What would kill or downgrade it?
4. What survives if it fails?

If any of those four are missing, the claim is not forbidden, but it is not yet load-bearing.

**Operational rule:** when a claim fails, prefer subtraction before invention. Shrink the claim first. Add replacement structure only after the downgrade path is explicit.
