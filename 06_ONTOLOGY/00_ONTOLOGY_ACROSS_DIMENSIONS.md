### Coordinate definition

The logarithmic coordinate makes the reciprocal symmetry explicit.

Let:

```text
x = ν > 0
s = log x
ν = eˢ
φ = e⁻ˢ
```

Then:

```text
φ · ν = 1
log φ + log ν = 0
(−s) + s = 0
```

So the Zero-Sum Resolution Equation becomes literal in the log register.

| Original register | Log register | Meaning |
|---|---|---|
| `1` | `s = 0` | multiplicative / logarithmic center |
| `x → 0⁺` | `s → −∞` | lower boundary / infinitesimal pole |
| `x → ∞` | `s → +∞` | upper boundary / infinite pole |
| `x ↦ 1/x` | `s ↦ −s` | reciprocal inversion becomes reflection |
| multiplication | addition | `log(ab) = log a + log b` |
| division | subtraction | `log(a/b) = log a − log b` |
| powers | scaling | `log(aⁿ) = n log a` |

This is the realignment:

```text
Zero is the additive center of x ↦ −x.
One is the multiplicative/logarithmic center of x ↦ 1/x.
```

So the center of this ontology is not absence as such. It is relation, identity, measure, and unity.

### Boundary discipline

Exact `0` is not inside the ordinary logarithm. The pole is reached only as a limit:

```text
x → 0⁺   maps to   s → −∞
x → ∞    maps to   s → +∞
```

Likewise, the emblem:

```text
⊙ = • × ○
```

may be read in the log register only with a frame-limit guard:

```text
for every finite R:    (−R) + (+R) = 0
therefore, in the balanced frame-limit:    the pole-pair returns the centre, ⊙
```

Do not write this as ordinary arithmetic `−∞ + ∞ = 0`. In extended arithmetic, `−∞ + ∞` is indeterminate. The framework uses the pole-pair as a symmetric limiting emblem, not as field arithmetic.
