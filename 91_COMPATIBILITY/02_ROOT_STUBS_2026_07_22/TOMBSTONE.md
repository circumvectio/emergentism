# TOMBSTONE — the nineteen zero-demand root stubs

Nineteen forwarding stubs were moved here from `01_EMERGENTISM/` root on
2026-07-22. **Nothing was deleted.** Each file is byte-identical to what stood
at root, moved with `git mv` so its history follows it.

## Why they moved — the measurement that changed the answer

An earlier pass in the same session counted inbound citations per stub, got
**2–33 each**, and concluded that none could move. That count was wrong in a
specific and instructive way: **it never asked who was citing.**

Five tidy passes between 2026-07-19 and 2026-07-22 produced their own audit
trail — L6 archive audits, contradiction firewalls, coherence passes, folder
censuses, `00_TIDY_PLAN_v0.1.md`. Every one of those documents names the stubs
it created or inspected. Re-counting with the machinery, the archives, the
dated handoff packets, and *other stubs* excluded:

| | stubs | live-doctrine citations |
|---|---|---|
| load-bearing, kept at root | **11** | 58 (all of it) |
| **zero live demand, moved here** | **19** | **0** |

`00_K5_REFUSALS.md` is the clean example: its citers were itself, four audit
packets about the tidy, a coherence audit, a folder census, and the tidy plan.
**The cleanup machinery cited its own output, and the count read that as
demand.**

## The cost, stated plainly

A stub only redirects from the path it sits at. Moving these means
`01_EMERGENTISM/<name>.md` **no longer resolves** for any citation *outside*
this repository — an old commit message, an external link, a bookmark. That
cost is real and was accepted knowingly: nothing inside the corpus needed them,
and the root cannot stay at 38 files forever because five tidy passes each
added stubs without ever removing one.

## Resurrection

Move any file back to `01_EMERGENTISM/` and it works again immediately —
nothing about it was rewritten. If external demand for one of these paths is
ever demonstrated, restoring it is a one-line act, not a reconstruction.
