# 95_COMPRESSED — Compressed Uplink Packets

**Evidence tier:** [S]
*Machine-compressed session packets for rapid agent ingestion.*

**Rosetta boundary:** [S/I] This README describes a lossy orientation layer. It does not [B] replace `60_SESSION_PACKETS/`, source-owner files, or current runtime-truth receipts.

> Compressed does not mean simplified. It means dense.

---

## What This Folder Is

A storage area for compressed (compiled) uplink packets. These are machine-generated summaries of session work, designed for rapid ingestion by agents joining mid-session or starting new sessions.

## Format

Compressed packets follow the naming convention:

```
{NNN}_{TOPIC}_{YYYY_MM_DD}.md
```

Where:
- `NNN` = packet number (sequential)
- `TOPIC` = brief topic descriptor
- `YYYY_MM_DD` = date

## Generation

Compressed packets are generated by:
1. The `compile_uplink.py` script (`09_TOOLS/01_SCRIPTS/`)
2. Manual compression by L6 Sādhu agents
3. Automatic compression at session end

## Usage

1. **Agent onboarding:** New agents read the most recent compressed packet instead of the full session log
2. **Cross-session continuity:** Compressed packets bridge sessions
3. **Cortex ingestion:** Compressed packets are indexed by Cortex for semantic search

## Relation to 60_SESSION_PACKETS/

- `60_SESSION_PACKETS/` = Full, uncompressed session packets (human-readable, detailed)
- `95_COMPRESSED/` = Compressed, machine-optimized packets (agent-readable, dense)

## ⚠️ Warning

[I] Compressed packets are **lossy**. They omit nuance, context, and dissenting views. For constitutional questions, descend to the full source packet in `60_SESSION_PACKETS/`.
