## 2. Translation Is Not Proof

Each cross-domain move must name its status.

| Status | Meaning |
|---|---|
| Unified | Same source kernel or directly derived structural relation |
| Translated | Same grammar carried into another register with bounded analogy |

## 4. Drift Checklist

Flag and repair these first:

- totalizing proof language: "proves the core state", "TOE", "final theory";
- bare `S²` physics overclaim, especially `SU(3)` or four-force closure;
- neuroscience overclaim, especially hemisphere identity or systemic awareness proof;
- old path roots such as `01_FOUNDATIONS/`, `03_EVIDENCE/`, `06_TRANSLATION/`, `07_DISSEMINATION/`;
- ambiguous `P`, `B`, `Φ`, `φ`, `ν`, `V`, or `η` notation;
- movement/gatekeeping language that turns routing into worth, access, or authority;
- generated or archived artifacts speaking as current source.

## 5. Folder Pass Order

For each folder and subfolder:

1. read the local README or index,
2. identify source owner and status,
3. separate active, generated, compatibility, and archive material,
4. repair the front door before deep files,
5. fix notation drift in active files,
6. repair only links whose targets are known,
7. leave historical audit reports as dated receipts unless they mislead active routing,
8. validate the local slice,
9. record unresolved questions for the next pass.
