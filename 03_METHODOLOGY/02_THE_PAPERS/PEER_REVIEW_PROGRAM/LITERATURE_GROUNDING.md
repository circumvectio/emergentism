# Literature Grounding Program

**Opened 2026-06-10.** A repeatable per-paper process: for every shard (R1–R9), find the related work, source and read the actual papers, reference them, and *refine the conjecture in light of what already exists* — shrinking false novelty and sharpening the genuine delta, then deriving a testable prediction. Modeled on the empowerment grounding (R6) the founder ran by hand.

## The process (per paper)

1. **Search** the related work (WebSearch; multiple angles).
2. **Identify** the canonical + most recent sources; record real URLs/DOIs.
3. **Source & read** — fetch open-access/arXiv full text; for paywalled, add to the **download list** for manual retrieval. *Ground claims only against text actually read.*
4. **Reference** — every external citation must trace to a fetched source. Memory-only citations are flagged `⚠ unverified` until checked. (This program exists because the corpus's risk was never fabricated citations — it was tier-inflating real ones.)
5. **Refine** — state honestly what the literature already covers (= not novel; cite, don't claim) and what remains the genuine delta.
6. **Predict / kill** — derive one testable prediction the corpus's version makes that the prior art does not, and the kill criterion if it fails.

## Discipline (carried from the program)

- The healthy outcome of a grounding pass is usually that the claimed novelty **shrinks**. That is the pass working, not failing (see R6: the teleology thread shrank to the conjugate geometry — a win, because it found the one defensible target).
- No tuning-to-confirm (cf. R9 refusal): if grounding shows a claim is already done or is refuted, record that, don't reframe it.
