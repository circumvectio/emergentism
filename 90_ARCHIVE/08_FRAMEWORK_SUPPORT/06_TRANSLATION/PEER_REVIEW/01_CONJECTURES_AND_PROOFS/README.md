# CONJECTURES AND PROOFS

## One Front Door for Peer Review

**Status:** Active  
**Date:** 2026-04-20  
**Purpose:** Provide one obvious lane-home for the framework's proof-facing and conjecture-facing peer review surfaces without moving or duplicating the existing review corpus.

> This folder is a **router**, not a second authority layer.
> The actual review materials remain in `../REVIEW_PROOFS/`, `../REVIEW_PACKETS/`, and `../REVIEW_PAPERS/`.

---

## What This Folder Is

This folder exists so a human reviewer or AI agent can answer one question quickly:

**"Where are the things that are being offered as proofs, and where are the things that are still conjectural?"**

It keeps those two surfaces adjacent without pretending they are the same.

---

## The Split

- **Proof-facing review surfaces** are the formal and structural packets that claim bounded mathematical or theorem-like validity.
- **Conjecture-facing review surfaces** are the papers, bridges, and frontier packets that remain `[C]` or materially depend on interpretive bridge work.

The point of this folder is not to flatten the distinction.
The point is to make the distinction impossible to miss.

---

## Read Order

1. Read [00_INDEX.md](00_INDEX.md).
2. If you are reviewing theorem-like material, go first to the **Proof Surfaces** table.
3. If you are reviewing frontier or bridge claims, go first to the **Conjecture Surfaces** table.
4. Then read the assigned packet only.

---

## Routing Rules

- If a document is presented as a proof, verify whether its current tier is actually `[S]`, `[A]`, or `[B]`; preserved packet labels may still use legacy `[E]`.
- If a document is presented as conjecture, keep it conjectural unless the owning source has already upgraded it.
- If this folder disagrees with [`../../../02_EPISTEMOLOGY/01_EVIDENCE_TIERS/00_THE_HONEST_POSITION.md`](../../../../../02_EPISTEMOLOGY/01_EVIDENCE_TIERS/00_THE_HONEST_POSITION.md), the Honest Position wins.
- If a packet mixes proof and conjecture, the reviewer should separate the claims before evaluating them.
