## The Cycle

### 1. Find (Targeting)
- **Action**: Locate the target.
- **Tools**: OSINT, Grep, Profilers.
- **Output**: A specific File, Function, or Bug ID.

### 2. Fix (Refining)
- **Action**: Pin the target down. Verify it is real.
- **Tools**: Reproduction Scripts, Test Cases.
- [B] **Output**: A failing test that proves the bug exists.

### 3. Finish (Execution)
- [I] **Action**: Eliminate the target.
- **Tools**: Code Editor.
- [B] **Output**: Green tests. The bug is dead.

### 4. Exploit (Intel Gain)
- [I] **Action**: What else can we learn?
- **Question**: "Is there another bug like this one elsewhere?"
- **Output**: A new search query (going back to *Find*).

### 5. Analyze (Assessment)
- **Action**: After Action Review (AAR).
- **Question**: "Why was it there? How did we miss it?"
- **Output**: `04_REALITY/INCIDENTS/POSTMORTEM-####.md`.

### 6. Disseminate (Sharing)
- **Action**: Tell the others.
- **Output**: Update `02_BLUEPRINT` or `RUNBOOKS`.
