# Evolution Protocol (Pattern Promotion)

**Type**: Normative Doctrine (Constitution)
**Status**: ACTIVE
**Trigger**: 3x Repeat of a Failure Mode or Success Pattern

---

## 1. The Purpose
To automate the transition from Episodic Memory (What happened?) to Semantic Memory (What is true?).
Principle: "The Definition of Insanity is doing the same thing twice and expecting a different result. The Definition of Evolution is prohibiting the failure mode after the third time."

**Rosetta boundary:** [I] This paper defines how patterns become proposed
invariants. It does not itself amend the constitution; [B] promotion requires
linked incidents/learnings, causality analysis, draft proposal, and ratification
receipts.

## 2. The Historian (Role)
The Historian is an autonomous function (Agent or Script) that audits the project's memory.
- **Cadence**: Weekly.
- **Source**: `07_WORK/LEARNINGS.md`, `04_REALITY/INCIDENTS/*.md`.
- **Mandate**: Identify Clusters (semantically similar events).

## 3. The Promotion Threshold (The Rule of Three)
A pattern is eligible for promotion to an Invariant (I#) [I] if and only if:
- **Frequency**: It has occurred > 3 times.
- **Causality**: The root cause is structural, not random.
- **Enforceability**: It can be written as a "MUST / MUST NOT" rule.

## 4. The Promotion Workflow

### Step 1: Detection
The Historian flags a cluster.
*Example: "Incident A (Timeout), Incident B (Timeout), Incident C (Timeout)."*

### Step 2: Drafting (The Mutation)
The Historian drafts a Proposed Invariant.
- **Format**: `PROPOSAL_I-XX.md`
- **Content**:
    - **Context**: Linked to the 3 incidents.
    - [I] **Proposed Rule**: "I-99: All external API calls MUST have a 5s timeout."
    - **Cost**: What is the friction of this rule?

### Step 3: Ratification (Natural Selection)
The User (Sovereign) or the Court of Owls reviews the proposal.
- **MERGE**: The rule is added to `./KERNEL_INVARIANTS.md`. The organism has evolved.
- **REJECT**: The cost of the rule exceeds its value.

## 5. Drift Correction (Pruning)
Evolution is also about shedding useless traits.
If an Invariant (I#) has not triggered a stop or saved a failure in 6 months, it is nominated for Deprecation.
