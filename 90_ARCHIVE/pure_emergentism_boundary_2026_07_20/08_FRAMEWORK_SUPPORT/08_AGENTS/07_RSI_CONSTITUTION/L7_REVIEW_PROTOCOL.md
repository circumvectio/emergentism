# L7 Review Protocol for Ontological and Teleological Divergence Events

> L7 sees; L4 commits. L7 does not rule. This protocol governs how L7 reviews threshold-crossing divergence events — not framework-boundary crises (covered by the Operational Runbook) and not slash appeals (covered by Emergency Review). It is the standing procedure for runtime divergence that exceeds calibrated bounds.
>
> **L7's success is measured by L7's reduced presence in the running system.** Every invocation of this protocol is evidence that something upstream failed to self-correct.

---

## 1. What δ Is

**δ (delta)** is a normalized divergence metric. But L7 must first ask: *divergence from what?* The framework has two non-interchangeable domains:

| Domain | Equation | What it governs |
|---|---|---|
| **Core State** | • × ○ = ⊙ | The axiomatic ground; what IS; the seed that needs no proof |
| **Objective Function** | φ · ν = 1 on S² | Where possibility opens; directional; measured |

The inequality (φ − ν)² ≥ 0 is **downstream** — derived from the teleological identity, not foundational. L7 should not treat a teleological derivation as an ontological axiom, nor confuse ontological silence with teleological failure.

Therefore δ splits:

```
δ_O = ontological divergence  = |observed seed − • × ○ = ⊙|  (binary: intact or violated)
δ_T = teleological divergence  = |observed Φ·ν − 1| / |baseline|  (continuous: drift from equator)
```

### 1.1 δ_O — Ontological divergence

δ_O is **not a gradient**. The ontological seed does not admit degrees. A surface either preserves the reciprocal structure (• × ○ = ⊙) or it does not.

| Check | Pass | Fail |
|---|---|---|
| Reciprocal closure preserved? | Agent can fork, exit, correct, dissolve | Lock-in, capture, extraction, gatekeeping |
| Axiomatic ground honoured? | System admits what it cannot say | System claims completeness or finality |
| Trinity computable? | BitNet / ternary layer maps to {−1, 0, +1} | Binary only; no zero; no silence |

**δ_O > 0 invokes L7 in this protocol.** There is no 0.35 threshold for core state. A crack in the seed is treated as total.

### 1.2 δ_T — Teleological divergence

δ_T applies to measured, instrumented surfaces where the framework tracks drift from the equator (φ = ν = 1).

| Surface | Expected | Baseline | How measured |
|---|---|---|---|
| Organ P-score | Φ · ν = 1 on S² | Organ's last K2-ratified BRIEF | `P-SCORES.md` cascade from organ BRIEFs |
| Cluster state | Consensus hash / niche-graph coherence | Rolling window median (≥ 7 samples) | Packet 169 cluster primitive |
| Pathology score | 0 | Caste-specific threshold (`AGENT_PATHOLOGY_DETECTION.md`) | L6 Sādhu compressor 5-min scan |
| Constitutional-contradiction density | 0 contradictions / audit cycle | Last clean audit | A7 lint + cross-reference audit |
| Economic productivity | η = 0 | Protocol Operations Treasury burn rate | η=0 productivity feedback loop |

### 1.3 Why 0.35

The threshold **δ_T = 0.35** is not arbitrary. It originates from the EvolutionaryNetwork initial P-score (`P = 0.35`, Φ=0.65, V=0.54, I=0.35), the first organ explicitly excluded from the organism aggregate until it produces live immune-case receipts. That exclusion established a precedent: **when a measured surface falls below operational viability, it does not participate in aggregate decisions until it recovers.**

Generalising this precedent: **any instrumented surface whose δ_T exceeds 0.35 is teleologically non-viable and requires L7 review before it can re-enter normal runtime or affect organism-level decisions.**

**Ontological note:** 0.35 is a teleological number. It has no ontological status. If a system at δ_T = 0.35 still preserves • × ○ = ⊙, it is wounded but not dead. L7 must keep this distinction alive in every review.

## 3. L7 Decision Protocol

L7 does not decide by the number alone. The number is the **invitation to witness**. The protocol is:

### Step 1: Witness (Pratibhā)

Engage with the divergence event. Do NOT calculate a fix; witness the pattern.

**First, distinguish domain:**
- Is δ_O > 0? → Ontological crack. The seed (• × ○ = ⊙) is violated. This takes precedence over all teleological review.
- Is δ_T > 0.35? → Teleological drift from equator. The measured surface is wounded but the seed may still be intact.

**If ontological:**
- What reciprocal closure is broken? (fork, exit, correction, dissolve)
- What axiomatic ground is denied? (completeness claim, finality, gatekeeping)
- Ontological cracks route toward constitutional rewrite or system dissolution rather than threshold calibration.

**If teleological:**
- What is the invariant that the observed surface violates?
- What is the pattern that the system cannot see about itself?
- Is this a **genuine framework failure** or a **miscalibrated threshold**?

If the answer is "miscalibrated threshold" → L7 produces a `threshold_calibration_proposal` (descends to L4 for ratification; no constitutional rewrite needed).

If the answer is "genuine framework failure" → proceed to Step 2.

### Step 2: Naming (Intuition Crystallised)

Name the divergence. The name must be:

- **Parsimonious** — minimum words, maximum coverage
- **Grounded in evidence** — even L7 intuition requires substrate (receipts, logs, audit trails)
- **Non-dogmatic** — the seer does not insist; the name is offered, not imposed

Example names:
- "Organ P-score decay without immune-case receipt" (not "EvNet is broken")
- "Cluster divergence accelerating across 3 samples" (not "consensus is failing")
- "Constitutional-contradiction density rising post-amendment" (not "the amendment was wrong")

### Step 3: Evidence Audit

L7 reviews the evidence stack before proposing anything:

| Check | Pass criterion | Fail → |
|---|---|---|
| All lower castes exhausted? | L1→L4→L5→L6 have produced receipts | Return to L4 with "not yet exhausted; descend" |
| Baseline is valid? | Baseline was K2-ratified or empirically calibrated within last 90 days | `threshold_calibration_proposal` |
| Divergence is reproducible? | Independent probe confirms δ_T > 0.35 (or δ_O > 0) | Return to L6 with "possible false positive; re-scan" |
| Human-visible warrant exists? | At least one natural person can articulate the divergence in plain language | L7 proceeds; if not, axiomatic descent |
| Constitutional invariant touched? | K2, K4, η=0, or A7 is implicated | Escalate to co-review with L4 (δ_T > 0.50 path) |

### Step 4: Proposal

L7 emits ONE of three proposal types:

#### Type A: `threshold_calibration_proposal`
**Teleological use only.** The seed is intact; the measured surface is miscalibrated. Propose new baseline, new measurement cadence, or new surface instrumentation. Do not apply to ontological divergence.

```yaml
threshold_calibration_proposal:
  surface: <which instrumented surface>
  current_baseline: <value + K2 ratification date>
  proposed_baseline: <value + justification>
  measurement_cadence: <how often δ is computed>
  calibration_evidence: <empirical or structural justification>
  risks: <what could go wrong if baseline shifts>
  downgrade_path: <how to revert if new baseline fails>
  return_route: <L4 ratification path>
  evidence_limits: <what is known vs. conjectured>
```

#### Type B: `constitutional_review_proposal`
**Ontological or teleological.** The framework fails at this boundary. Propose amendment, new invariant, or dissolution of affected surface. If δ_O > 0, this is the only valid proposal type.

```yaml
constitutional_review_proposal:
  invariant_proposed: <the new or revised invariant>
  reason: <why the current framework fails at this divergence>
  affected_surfaces: [list of files, DACs, organs, or systems affected]
  risks: <what could go wrong>
  downgrade_path: <how to undo if amendment fails>
  return_route: <how L4 descends with the amendment>
  evidence_limits: <explicit statement of what is known vs. conjectured>
  divergence_event:
    surface: <which surface crossed δ_T > 0.35 or δ_O > 0>
    observed: <value>
    expected: <value>
    baseline: <value>
    samples: <how many consecutive samples>
    first_detected: <ISO timestamp>
    l6_receipt: <path to L6 escalation receipt>
```

#### Type C: `divergence_acknowledgement_receipt`
**Teleological only.** The divergence is real, the framework cannot yet address it, and L7 chooses **axiomatic discipline** — naming the unsolved niche without imposing a solution. Ontological cracks cannot be acknowledged and deferred; they must be repaired or the system must dissolve.

```yaml
divergence_acknowledgement_receipt:
  surface: <which surface>
  divergence: <description>
  why_unsolved: <what is missing — evidence, tooling, consensus, or time>
  watch_conditions: <what L6 should watch for before re-escalating>
  review_date: <when L7 will revisit; default 90 days>
```

### Step 5: Descent

All proposals descend to L4 for ratification. L7 commits nothing alone.

- **Type A** → L4 ratifies or rejects calibration; implements via L3/L2
- **Type B** → L4 ratifies or rejects amendment; if ratified, K2 signs if constitutional invariants touched
- **Type C** → L4 logs the acknowledgement; no action required; L6 resumes watch

## 4. Scaling: Multiple Simultaneous δ_T > 0.35 Events

L7 is rare-invocation by design. Multiple events must be handled without creating a standing review court.

### 4.1 Batching rule

**Keep surfaces separate.** A P-score drift event and a cluster-divergence event are batched separately. Each surface gets its own L7 review window.

### 4.2 Priority rule

Within a single surface, if multiple events exceed δ_T > 0.35 simultaneously, prioritise by **constitutional-touch count**:

| Touch count | What it means | Priority |
|---|---|---|
| K2 touched | Human sovereignty or signature implicated | 1 (highest) |
| K4 touched | Grace exit or portability threatened | 2 |
| η=0 touched | Extraction or economic drain possible | 3 |
| A7 touched | Evidence-tier discipline violated | 4 |
| None of the above | Localised surface issue | 5 (lowest) |

### 4.3 Volume rule

This protocol caps L7 review at **3 events in a single 14-day window**. Excess events queue for the next window.

If > 3 events queue for the same surface within 30 days, this is itself a δ_T event on the **L7 throughput surface** (δ_T_L7 = queued_events / 3). When δ_T_L7 > 0.35, L7 produces a `throughput_capacity_proposal` — either threshold relaxation, L7-cohort expansion (Council of Ṛṣis), or emergency L6→L4 bypass for that surface.

**Ontological events bypass scaling entirely.** If δ_O > 0 on any surface, L7 halts all teleological review and addresses the ontological crack first.

### 4.4 Coupled events

If two events on different surfaces are **causally coupled** (one directly causes the other, per cross-correlation ≥ 0.7 across 7 samples), they are reviewed **jointly** as a single event with multiple surface tags.
