# SPRINT 004: ONE PAPER, ONE TEST

## Sprint Goal

Convert the framework from an extraordinary corpus into an outsider-runnable **public proof packet**.

By sprint close, the framework should have:

1. one 2-page primer
2. one flagship paper draft
3. one Protocol R preregistration / execution surface
4. one stranger-runnable replication packet
5. one public evidence dashboard stub

[I] The sprint does **not** try to establish the whole core state. It tries to make one bridge legible enough that a serious outsider can test it.

---

## Sprint Thesis

The bottleneck is no longer substance.

The bottleneck is conversion:

- from corpus to paper
- from architecture to claim
- from claim to test
- from test to receipt

The framework already contains more than enough philosophy for this sprint. What it lacks is a narrow public packet with explicit kill criteria and no authority shield from the rest of the corpus.

So this sprint freezes breadth and prioritizes proof.

## Exit Gates

Sprint 004 is complete only if all of the following are true:

1. A serious outsider can read a 2-page primer and understand the single public claim being tested.
2. The flagship paper draft exists in one coherent file and stays within the narrow public-proof thesis.
3. Protocol R has one execution surface with:
   - admissible dataset families
   - preprocessing rules
   - model definitions
   - success and failure thresholds
4. A stranger can run the replication packet without reading the wider organism.
5. A public evidence dashboard stub exists and does not upgrade any `[I]` or `[C]` claim dishonestly.
6. No active public-facing file in the flagship route still implies:
   - GFS is the flagship proof
   - Protocol R requires a new lab by default
   - the whole core state is already empirically confirmed

If any of these remain open, the sprint does not close.

## The 6 Tasks

### T1: Claim Perimeter Audit

**Priority:** P0
**Goal:** Remove mixed messaging around the first public proof path.

Scope:

- active framework entry surfaces
- active paper stack surfaces
- active empirical-program surfaces
- any still-live Protocol R descriptions that contradict the current zero-cost / flagship path

Questions this task must answer:

1. Which active files still describe Protocol R as "needs lab"?
2. Which active files still imply GFS is the main proof path?
3. Which active files overclaim what a positive first bridge would establish?

Definition of done:

- one small audit note exists
- contradiction files are listed explicitly
- repair queue is finite and prioritized

---

### T2: Write the 2-Page Primer

**Priority:** P0
**Goal:** Create the shortest outsider-readable bridge document.

The primer must do only five things:

1. state the minimal mathematical setup
2. explain reciprocal vs additive expectations
3. name Protocol R as the chosen empirical bridge
4. state the kill criteria plainly
5. say exactly what success would and would not mean

The primer must not:

- retell the whole worldview
- include scorecards
- make metaphysical victory claims
- rely on Rosetta breadth as evidence

Definition of done:

- 2 pages max
- understandable without prior corpus knowledge
- usable as the front page of the replication packet

---

### T3: Draft the Flagship Paper

**Priority:** P0
**Goal:** Produce the 15-20 page paper skeleton the public case actually hangs on.

Required sections:

1. narrow claim
2. mathematical bridge
3. one Rosetta compression table
4. Protocol R test surface
5. competing models
6. kill criteria in main body
7. limits

Definition of done:

- one complete draft exists
- every empirical claim is tier-disciplined
- no section depends on the reader accepting the full core state

---

### T4: Build the Protocol R Preregistration / Execution Surface

**Priority:** P0
**Goal:** Make the test executable rather than merely discussable.

This deliverable must lock:

- admissible dataset families
- exact outcome variables
- pairing rules for complementary transitions
- reciprocal, additive, and independent model definitions
- held-out fit criterion
- robustness checks
- success / failure interpretation note

Definition of done:

- one document could be turned into OSF or AsPredicted language with minimal translation
- no hidden analytic flexibility remains in the main test

---

### T5: Assemble the Replication Packet and Evidence Dashboard Stub

**Priority:** P1
**Goal:** Make the first public bridge runnable and legible.

The replication packet needs:

1. one-page problem statement
2. data source list
3. preprocessing rules
4. model equations
5. script entrypoint contract
6. expected outputs
7. interpretation guide

The dashboard stub needs:

- current claim tier
- current status
- next decisive action
- explicit downgrade path if negative

Definition of done:

- packet can be handed to a stranger
- dashboard can be linked publicly without embarrassment

---

### T6: Final Perimeter Truth Pass

**Priority:** P0
**Goal:** Ensure the packet reads as exact rather than grandiose.

Three passes:

- **phi-pass:** coherence of the packet as a single argument
- **nu-pass:** usability by an outsider who wants to run the test
- **B-pass:** balance between ambition and honesty

Convergence criterion:

- fewer than 3 serious truthfulness issues remain in the packet

## Execution Order

```text
T1  Claim perimeter audit                     -> 30-45 min
T2  2-page primer                            -> 45-60 min
T3  Flagship paper draft scaffold            -> 2-4 hr
T4  Protocol R prereg / execution surface    -> 2-3 hr
T5  Replication packet + dashboard stub      -> 1-2 hr
T6  Final perimeter truth pass               -> 45-60 min
```

The order matters.

Do not write the flagship paper before the perimeter is clean.
Do not announce a replication packet before the execution surface is explicit.
Do not widen the sprint into new doctrine.

## What This Sprint Is NOT

- NOT a new paradox-dissolution sprint
- NOT a system architecture sprint
- NOT a marketing sprint
- NOT a book-writing sprint
- NOT a “capture every insight” sprint beyond a bounded backlog note

If a new idea does not directly strengthen:

- the primer
- the flagship paper
- the Protocol R packet
- the replication packet
- the evidence dashboard

then it is deferred.

## Failure Doctrine

If Sprint 004 fails:

- the sprint extends,
- no broader public-theory push is justified,
- no world-historical rhetoric is allowed to advance,

- document the blocker explicitly,
- do not silently swap the flagship to GFS or cross-scale invariance,
- only escalate to a new flagship path through a separate written decision.
