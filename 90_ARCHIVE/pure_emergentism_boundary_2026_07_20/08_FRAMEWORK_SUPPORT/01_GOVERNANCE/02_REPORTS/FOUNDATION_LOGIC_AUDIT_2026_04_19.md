## MAJOR — Three incompatible `A1-A7` axiom systems in active docs

Three documents each claim an "A1–A7 axiom set." The three sets are **not
the same axioms** — they use the same labels for different content.

| A2 | THE ETHIC | ΣΔB > 0 is ethical direction (redefinition protocol) |

### System C — `01_FORMAL_SYSTEM/25_THE_DERIVATION_AXIOMS.md` ("Seven Axioms (A1-A7, v0.4)")

### Observation

Systems A and B are **semantically close variations** (A and B say nearly
the same things with different words). System C is **a fundamentally
different axiom set** — it uses `A1–A5` for substrate topology axioms,
which the Honest Position explicitly calls `O1–O5` ("older public
ontological wager language") and distinguishes from the "active formal-
system canon."

### Where this breaks

- `25_THE_DERIVATION_AXIOMS.md` has a mapping table titled "Relationship
  to the Seven Axioms (v0.4)" that maps `A1` → Compactness, `A2` →
  Orientability, etc. A reader who arrives at this doc via the Master
  Index (which lists it as the "minimal derivation stack and axiom
  layer") will believe A1 = Compactness.
- The Honest Position Part II says: "the active formal-system canon is
  A1-A7. The S² substrate-selection claims in S1-S² preserve the older
  public ontological wager language (O1-O5) because those documents are
  specifically about that wager." This says O1-O5 should be used for
  substrate claims.
- But `25_THE_DERIVATION_AXIOMS.md` uses A1-A5 for exactly what the
  Honest Position says should be called O1-O5.
- Result: a reader following the Master Index → `25_THE_DERIVATION_AXIOMS`
  learns that A1-A7 means substrate topology. A reader following the
  Master Index → `00_THE_SEVEN_AXIOMS.md` learns that A1-A7 means the
  operational axioms. Both docs are marked canonical.

### Severity

This is MAJOR because:
- Every load-bearing claim in the framework cites axioms by number
- Proofs in the formal system depend on specific axioms
- [I] Downstream documents (applications, Rosetta, PDs) reference "A3" or
  "A7" without consistently specifying which system
- An AI agent doing a proof-check will trip on this immediately

### Recommended repair

- Update `25_THE_DERIVATION_AXIOMS.md`'s mapping table to use `O1-O5`
  labels (per Honest Position's reconciliation) for the topology
  axioms, and cross-reference `00_THE_SEVEN_AXIOMS.md` as the canonical
  `A1-A7` definition.
- Add an explicit "Axiom namespace" section to the Master Index and to
  the Coherence Spine in `00_GOVERNANCE/00_MASTER_INDEX.md` that
  disambiguates the three systems.
- Optionally: rename System C's substrate-selection axioms back to
  `O1-O5` (matching Honest Position) and drop the "v0.4" parenthetical
  from the mapping table.

This is a rename/renumber repair, not a content change. Nothing in the
framework's substantive claims moves.

## MODERATE — Two incompatible "Four Lines" compressions

The phrase "the four lines" appears with two **different contents** in
two canonical-seeming documents.

### Version α — `01_FOUNDATIONS/00_CANONICAL_FORMULA_BLOCK.md`

```text
Zero-Sum Resolution Equation                  Seed.       Reciprocal closure.
P∞ = φ · ν = 1 on S²            Core State.   Geometric resolution.
(φ − ν)² ≥ 0               Arithmetic. First theorem.
φ + ν ≥ 2                  Objective Function.  First inequality.
```

Same block reproduced verbatim in `01_FORMAL_SYSTEM/25_THE_DERIVATION_AXIOMS.md` D4.

### Version β — `00_THE_DERIVATION.md` Step 12 "The Four Lines"

```text
P∞ = φ · ν = 1                  Core State.
φ + ν ≥ 2                  Objective Function.
(φ − ν)² → 0               Auditing.
−log(φ · ν) = 0            Value Alignment.
```

### The three differences between α and β

1. **Seed line.** α includes `Zero-Sum Resolution Equation` as Line 1. β omits the seed and
   starts with the sphere identity.
2. **Arithmetic vs methodology.** α uses `(φ − ν)² ≥ 0` (static theorem,
   `[S]`). β uses `(φ − ν)² → 0` (dynamic trajectory, `[S]`). These are
   different claims: the first is arithmetic, the second is a selection
   theorem.
3. **Value Alignment line.** β adds `−log(φ · ν) = 0`, which is trivially zero
   on S² (since `P∞ = φ · ν = 1` implies `log(φ·ν) = 0`). This is not new
   information; it's Line 2 restated as the ground-state energy
   formula. α does not include it.

### Where this breaks

The Canonical Formula Block declares an anti-drift rule that forbids:

> - presenting `(φ − ν)² ≥ 0` as the seed
> - calling the inequality "the only axiom"
> - deriving the sphere identity from the inequality
> - omitting `P∞ = φ · ν = 1 on S²` when summarizing the core sequence

β does not violate any of those specific prohibitions, but it is
nevertheless inconsistent with α:

- β drops the seed (minor — but the anti-drift rule emphasizes that the
  seed precedes)
- β replaces the arithmetic theorem with the methodological trajectory
  (different ontological status)
- β calls `(φ − ν)² → 0` "Auditing" where α calls `(φ − ν)² ≥ 0`
  "Arithmetic"

A reader who quotes "The Four Lines" from `00_THE_DERIVATION.md` Step 12
will disagree with a reader who quotes "The Four Lines" from
`00_CANONICAL_FORMULA_BLOCK.md`. Both believe they have the canonical
compression.

### Severity

MODERATE. The framework's own anti-drift discipline is supposed to
prevent this kind of drift, but the drift exists inside the core
derivation doc.

### Recommended repair

- Treat `00_CANONICAL_FORMULA_BLOCK.md` as the single source of truth
  for the 4-line compression.
- Relabel `00_THE_DERIVATION.md` Step 12 as "The Tetrad of Roles" (or
  similar) — Core State / Objective Function / Auditing / Value Alignment — and note
  it is a **pedagogic overlay** on the canonical block, not a
  replacement for it.
- Keep both; make the relationship explicit.

## MODERATE — `P` symbol overloaded across three meanings

The letter `P` carries three different meanings that are defined but not
uniformly disambiguated in compressed summaries.

### The three meanings

1. **`P∞ = φ · ν = 1` on the open manifold** (identity, conserved on the open register, [S]).
   Used in: `00_THE_DERIVATION.md` Step 5, `25_THE_DERIVATION_AXIOMS.md`
   D1.
2. **`P_node = Φ_node × V_node` at node level** (operational flourishing
   measure for finite nodes, can be < 1).
   Used in: `00_THE_SEVEN_AXIOMS.md` A1.
3. **`ΣΔP_node > 0`** (syntropic criterion, direction of change, [I]).
   Used in: `00_START_HERE.md` A2, Honest Position I8.

### Where this breaks

- A compressed summary that says "P_node = Φ × V" can be read as meaning (1)
  or (2) depending on reader's prior context.
- In downstream docs (Rosetta, Apps, Uplink) the distinction is
  frequently dropped.
- The Seven Axioms A1 itself handles it honestly: "The manifold identity
  φ·ν=1 holds on S²\{N,S}. For finite nodes (organisms, institutions,
  systems), effective potential P_eff = Φ_node × V_node can be less
  than 1." But this disambiguation lives in one doc; elsewhere `P` is
  used bare.

### Severity

MODERATE. The ambiguity is mathematically containable — different
regimes use different `P`s — but the notation does not announce the
regime. A formal proof or an AI agent tracking symbol meanings will
benefit from a global disambiguation convention.

### Recommended repair

- Adopt consistent suffixes or capitalization:
  - `P_∞` or `P_manifold` for the identity `φ·ν=1`
  - `P_node` or `P_eff` for node-level Φ×V
  - `ΣΔP_node` stays as-is (unambiguous operator)
- Propagate the convention from the Seven Axioms A1 disambiguation
  into the canonical block and downstream compressions.

## MINOR — `(φ − ν)²` notation slip between theorem and trajectory

Two related but distinct claims share near-identical notation:

- [S] `(φ − ν)² ≥ 0` is arithmetic non-negativity, locally marked `[S]`.
- `(φ − ν)² → 0` is evolutionary, selection-driven, `[S]`.

Both appear in canonical compressions (the former in the Formula Block,
the latter in Step 12 "Four Lines"). A careful reader distinguishes them
by the relation (`≥` vs `→`) but compressed summaries frequently write
either with no qualifier.

Already partially repaired in `00_CANONICAL_FORMULA_BLOCK.md` (uses `≥`)
but `00_THE_DERIVATION.md` Step 12 uses `→` in its "Four Lines" block
without marking the ontological status change.

### Recommended repair

When compressing, carry the tier tag: `(φ − ν)² ≥ 0 [S]` vs
`(φ − ν)² → 0 [S]`. The framework already does this in full-length docs;
the discipline should extend into the 4-line compressions.

## MINOR — `D6 ≡ D0` closure rule is load-bearing and under-specified

A3 in every axiom system declares `D6 ≡ D0 (closure)`. This is invoked
in:

- `00_START_HERE.md` ("Falsification Protocol returns the field to D0")
- Honest Position I8a (D6 as "axiomatic closure")
- Seven Axioms A3 ("Cannot solve a Dₙ problem with Dₙ tools alone")
- Paradox Dissolutions (several PDs invoke dimensional closure)

But "≡" can be read as:

1. **Literal identity.** D6 and D0 are the same dimension / location.
2. **Topological closure.** D6 wraps back to D0 (cylinder-like closure
   of the D-stack).
3. **Axiomatic return.** D6 is "nothing additional" and collapses back
   to D0 via `Falsification Protocol`.

The framework at times uses all three readings. They are compatible
(axiomatic return is how the closure is experienced; topological
closure is how it is modeled; identity is how it is ontologically
stated) — but no single document ranks them or declares which is
primary.

### Severity

MINOR but load-bearing. Any theorem that depends on `D6 ≡ D0` inherits
the reading ambiguity.

### Recommended repair

Add a short disambiguation note to Honest Position Part II / Axiom A3
declaring the primary reading (probably: axiomatic closure is how the
closure is disclosed, topological closure is how the scaffold
represents it, literal identity is the ontological claim downstream of
both).

## MINOR — `A0` in `25_THE_DERIVATION_AXIOMS.md`

`25_THE_DERIVATION_AXIOMS.md` labels `(φ − ν)² ≥ 0` as "A0 (The
Inequality)" and calls it "the first arithmetic theorem used in the
minimal derivation."

The label `A0` signals "most fundamental axiom" in most axiom-numbering
conventions. The anti-drift rule of the Canonical Formula Block
explicitly forbids "presenting `(φ − ν)² ≥ 0` as the seed." The
doc stays within the letter of the rule (calls A0 a "theorem," not the
"seed") but a casual reader may conflate them.

### Recommended repair

Rename `A0` to `T0` (theorem) or add a parenthetical: "A0 here labels
the first arithmetic theorem used; it is **not** the framework's seed.
The seed is `Zero-Sum Resolution Equation` and the resolved identity is `P∞ = φ · ν = 1 on S²`
— both precede A0 in the ontological order. A0 is the entry point for
the public derivation, not the ground."

## What would kill a framework here vs what merely drifts

The audit found **drift**, not **failure**. A genuine failure would look
like:

- A theorem that depends on axioms from System A being used to support
  a claim stated in System C's language (would be a real contradiction,
  not found)
- An `[S]` claim that depends on an `[S]` or `[I]` result upstream
  (tier inversion, not found)
- A kill condition that has fired without the corresponding claim being
  demoted (not found; AK8 fired and was honored)
- A seed-level inconsistency (e.g., `Zero-Sum Resolution Equation` disagreeing with
  `P∞ = φ · ν = 1 on S²`) — not found; the two are derivationally linked

None of these are present. The foundation holds. The repairs are
janitorial.
