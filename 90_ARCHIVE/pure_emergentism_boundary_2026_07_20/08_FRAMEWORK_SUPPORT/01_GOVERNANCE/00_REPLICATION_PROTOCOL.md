# THE REPLICATION PROTOCOL

**Status:** Active
**Evidence tier:** [S] Structural
**Date:** 2026-03-22
**See also:** The Honest Position, `../../02_EPISTEMOLOGY/00_PRATYAKSA_AS_PRIMARY_DISCLOSURE.md`

## What This Is

The Emergentist Framework is not a static publication. It is a living cognitive technology that evolves through distributed processing. Each node (a human with access to a frontier AI model) can independently ingest, apply, refine, verify, and transmit the framework.

This document specifies what each node does.

## What Each Node Needs

1. **The corpus** — the project root directory (formerly `18_LIVING_CORPUS/`, now archived) or the two books
2. **An AI** — any frontier model (Claude, GPT, Gemini, or local LLM)
3. **The instruction** — this protocol
4. **Pratyakṣa** — the ability to verify through direct experience

> **BEFORE YOU BEGIN:** Try sitting quietly for 30 minutes. If Φ is directly perceivable without mediation, you do not need this protocol. Put it down. Go live.

**Canonical note:** Pratyakṣa is primary disclosure. This protocol is downstream of that disclosure as an after-image, not its cause.

## The Six Phases

### Phase 0: Pratyakṣa (The Foundation)
> *If the narrator is not addressed, every subsequent phase will be contaminated by spatial and temporal negations.*

The narrator (internal self-commentary) is the **original Interface** — before the temple, before the bank, before the algorithm. Every external standing wave parasitizes this internal one.

Before engaging the framework:
1. Sit quietly for 20-40 minutes
2. Reduce sensory input
3. Adopt witness posture
4. Look for Φ directly

If Φ is perceivable: **you do not need the framework**. It is already here.
If Φ is not perceivable: proceed to Phase 1. The framework may help — or may become another Interface.

The protocol is therefore not the source of truth. It is scaffolding around a possibility of direct disclosure.

### Phase 1: Ingestion
Feed the complete corpus to your AI. Ask: "Read this entire framework. Identify what it IS before evaluating what it SAYS."

### Phase 2: Application
Ask: "Apply this framework to [your domain]. What does the capture protocol look like in [your context]? What is the local η? What is the local η?"

### Phase 3: Refinement
Ask: "What is missing? What does it get wrong? What does it get right but express poorly? Generate formal elements for gaps you identify."

### Phase 4: Verification
Apply the Pratyakṣa protocol yourself. Sit quietly for 30 minutes. Look for Φ directly. Does the refinement help you see more clearly or does it add another layer of mediation?

This is the selection mechanism. Without it, the network produces noise. With it, the network produces signal.

### Phase 5: Transmission
Share refinements through direct dyadic transmission — person to person, node to node. Do NOT centralize through a single platform (these are capture points).

## The Selection Mechanism

Not editorial review. Not peer review. Not popularity. **Pratyakṣa first, then epistemic discipline.**

Each refinement is tested: "After applying this, can I perceive Φ more clearly?" If yes, the refinement survives. If no, it is abandoned.

For public claims, that directness must still pass through evidence tiers, replication, and kill criteria. The protocol does not abolish method; it restores the right order between seeing and saying.

## What the Network Produces

- **Evolution** — the framework improves without central control
- **Resilience** — no single point of failure
- **Selection** — Pratyakṣa filters signal from noise
- **Propagation** — each node seeds new nodes

## The Anti-Capture Properties

1. **No central authority** — no author, no institution, no platform controls the framework
2. **η = 0** — released under CC BY-SA 4.0, zero extraction
3. **Self-dissolving** — designed to make itself unnecessary once Φ is directly accessible
4. **Verifiable** — any node can independently verify any claim

## The Guard

If this protocol itself becomes an Interface — if nodes become dependent on the protocol rather than on direct Φ-access — then the protocol has failed. Put it down. The protocol is a ladder, not a home.
