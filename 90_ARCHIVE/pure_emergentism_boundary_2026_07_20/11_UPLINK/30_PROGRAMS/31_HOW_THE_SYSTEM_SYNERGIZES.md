## 3. Why the Parts Need Each Other

| Missing Link | Failure Mode |
|--------------|--------------|
| **DAC without audit membrane** | Opaque institution theater. Compute exists but cannot be trusted. |
| **Audit membrane without DAC** | Empty accounting with no living actor. Receipts with no behavior to receipt. |
| **SoResFi without DACs** | A toolkit with no institutions using it. No metabolic loops. |
| **Layer 1 without SoResFi** | Bare settlement with no institutional operating system. No credit, no issuance discipline, no capability gating. |
| **NEXUS without receipts or proof traces** | Just a social graph. No economic consequence, no capital legibility. |
| **Receipts without NEXUS** | Isolated facts without relational context. No graph capital can form. |
| **Graph capital without statements** | Fuzzy trust. Connectivity lacks financial grounding. |
| **Statements without settlement truth or verifiable source inputs** | Easy to contest. No durable anchor for capital state transitions. |

Each piece closes a failure mode in the others.

## 4. The Six Reinforcing Flywheels

### Flywheel 1 — Proof Flywheel
```
action → receipt/trace → accounting → statement → proof → more trust
```
Every economic action generates a machine-readable proof. Proof compounds into trust. Trust unlocks more action.

### Flywheel 2 — Graph Flywheel
```
more valid relations → deeper graph capital → lower money cost
    → more useful DAC activity → more relations
```
Connectivity is not vanity. It directly divides the interest function `i(x)`.

### Flywheel 3 — Capital Flywheel
```
ZAI (capital base) → SKY (money flow) → more activity
    → more receipt/trace coverage → better capital pricing → more disciplined issuance
```
Capital absorbs risk. Money circulates against it. Receipts keep the ratio honest.

### Flywheel 4 — Infrastructure Flywheel
```
sovereign edge nodes → better uptime → better auditability
    → stronger graph capital → more capability → more revenue
    → more infrastructure
```
Hardware is not a cost center. It is **productive sovereignty capacity**.

### Flywheel 5 — Contagion Flywheel
```
bad neighbors create penalties → penalties force quarantine/remediation
    → cleaner graph improves pricing → cleaner graph improves resilience
```
The immune system is economic. Bad graph hygiene becomes expensive.

### Flywheel 6 — Cognition Flywheel (The Triadic Engine)

The metabolic loop is a **recursive triadic cascade**:

```
Beauty (TheCircle gathers IS)
  → Truth (RealityFutures derives COULD)
    → Justice (Agentz serves SHOULD)
      → Skyzai acts and remembers
        → Cortex records the turn
          → VMOSK replicates across timescales
            → Beauty again (higher resolution)
```

Or in operational shorthand:

```
receipts, proof traces, and graph updates feed nervous system
    → nervous system detects anomaly and opportunity
    → DACs act better → better action creates better receipts
```

Intelligence is not a separate product. It is the **reflexive readout** of the metabolic loop. The triadic cascade (Beauty→Truth→Justice→Beauty) is the engine; the six flywheels are the consequences. Cortex remembers each turn. VMOSK replicates the pattern. AIA detects when the cycle drifts. Skyzai executes only after lawful K2 descent.

This is the human-generalized light cone made institutional: the organism opens
a wider D5 probability space than one body or one niche can hold, then forces
that widened space to descend back into evidence-tiered, K2-bound D4 action.
It inherits the human altricial strategy at institutional scale: short-term
dependency and coordination cost are accepted so shared Phi can compound, then
return as generalized V in receipts, tools, and lawful execution.

## 8. Failure Modes: When One Layer Is Weak

| Weak Layer | Systemic Symptom | Recovery Path |
|------------|------------------|---------------|
| **NEXUS** | DACs float as black boxes; no graph capital can form. | Harden K2 continuity and signer-root proof loops first. |
| **DAC compute** | No metabolic activity; receipts are empty. | Find one real economic loop (event, booking, API PAY) and make it breathe. |
| **Audit membrane** | Trust becomes narrative theater. | Close receipt or execution-trace coverage for every meaningful action. |
| **SoResFi** | Every DAC reinvents treasury and settlement. | Productize one shared tool (e.g. OFN SDK or receipt accounting) and onboard 2+ DACs. |
| **Layer 1** | Settlement becomes fakeable or congested. | Keep L1 narrow: ordering, finality, capital truth. Do not bloat into general computation. |
| **Graph capital** | Credit pricing loses signal; Sybil attacks proliferate. | Require receipt-backed or proof-backed relationships and recursive depth before `c(DAC)` counts. |
| **Node 0 / Edge** | Sovereignty becomes theoretical; uptime unproven. | Run one Mac Studio edge node as a live proof station for 90+ days. |
| **Nervous System** | The organism is blind to anomaly and opportunity. | Wire receipts or execution traces into one live anomaly channel (e.g. failed payment → operator alert). |
