## Canonical Contradictions Requiring Reconciliation

### 1. ZAI Demurrage
**Current problem:** documents describe mechanical idle-balance decay and redistribution, while current implementation does not support that behavior as a live token reality.

**Canonical options:**
1. implement a real demurrage mechanism,
2. move demurrage into a wrapper/staking layer,
3. remove demurrage from active monetary claims.

**Current status:** FROZEN

---

### 2. SKY Interest Flow
**Current problem:** documents describe interest accrual and routing to an AMM/protocol flow, while implementation does not yet support that as live reality.

**Canonical options:**
1. implement actual cost-of-capital and fee-routing mechanics,
2. redefine SKY economics honestly as non-interest-bearing in current state,
3. remove protocol-interest-routing claims until implemented.

**Current status:** FROZEN

---

### 3. Lombard Bridge
**Current problem:** the same name appears to refer to materially different credit products across canon/spec/code.

**Canonical options:**
1. choose one product definition and keep the name,
2. split the products and rename the non-canonical variants,
3. freeze the term entirely until one credit model survives.

**Current status:** CANONICAL DECISION REQUIRED

---

### 4. LP-100 Supply Invariant
**Current problem:** fixed-supply rhetoric and mint-capable implementation are in conflict.

**Canonical options:**
1. make supply actually fixed and remove minting path,
2. admit capped inflation/dilution honestly,
3. rewrite K1-related claims to match implemented mechanics.

**Current status:** FROZEN

---

### 5. Grace Exit
**Current problem:** strong redemption/NAV language appears ahead of reserve, payout-asset, and solvency mechanics.

**Canonical options:**
1. fully fund and implement Grace Exit as promised,
2. redefine it as a target state rather than current guarantee,
3. suspend the promise until unencumbered-asset truth and payout mechanics are real.

**Current status:** FROZEN

---

### 6. K2 Sovereign Signature Guarantees
**Current problem:** K2 is constitutionally central, but strong human-signs-every-action guarantees exceed current implementation certainty.

**Canonical options:**
1. implement full cryptographic verification end-to-end,
2. clearly distinguish staging/demo/partial from true sovereign execution,
3. freeze strong guarantee language until verification is real.

**Current status:** FROZEN

## Canonical Financial Decision Stack

Before any financial/execution launch language is reactivated, each claim must pass all four layers:

1. **Product identity coherence**
   - one name = one product
   - no multiple incompatible instruments sharing one label

2. **Contract/mechanic coherence**
   - code actually does what the docs claim
   - no metaphor presented as mechanism

3. **Balance-sheet coherence**
   - exits/redemptions are actually fundable
   - liabilities are not described without reserves/mechanics

4. **Constitutional coherence**
   - sovereignty language matches signature/enforcement reality
   - no “human signs every action” overclaim if bypasses/stubs remain
