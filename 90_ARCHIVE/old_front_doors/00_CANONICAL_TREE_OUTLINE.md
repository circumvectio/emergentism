## Numbering conventions — INTENTIONAL, not drift (read before "fixing" anything)

Per the repo `CLAUDE.md`, the following are **canonical** and must **not** be normalised:

- **`00_*` is a semantic flag**, not an ordinal — primitive/foundational content. **Multiple `00_*`
  files or dirs at the same depth is by design** (e.g. 05_COSMOLOGY's many `00_*` docs).
- **Date-format variance is provenance encoding:** `YYYY_MM_DD` = native canon · `YYYY-MM-DD` =
  external import · `YYYYMMDD` = pipeline artifact. Do not normalise dates.
- **Case-suffixes carry meaning:** lower-case (`103a`) = machine-readable sidecar; UPPER-case
  (`175A`) = variant content.
- **Filename collisions opening with "Compatibility stub. The canonical home … moved to …" are
  intentional** Rosetta-reorg compat-stubs, not duplicates.
- **The -ology lane order (01–07) is canonical.** Cross-lane reuse of `THE_DERIVATION` (teleology owns
  the proof corpus; methodology holds the spine pointer) is intentional.
- **Documented gaps are not drift:** `02_THE_PAPERS` letter gaps (A–P, T, U; R via Paper J);
  `03_FORMAL_SYSTEM` `00,00,00 → 08` supersession; `PARADOX_DISSOLUTIONS` PD_01/02/03/17 (tombstoned
  in `PD_00_INDEX.md`); `11_UPLINK/50_AUDITS` interleaved-date ordinal doubling; `70_EXTRACT_NOW`
  sparse R-/S- IDs (`00_MANIFEST.md` is authoritative).

Genuine drift (typos, mis-styled filenames, stale maps, broken pointers) is tracked and repaired
under the tree-tidy audit; this section exists so it is never confused with the conventions above.
