# 09_TOOLS — Emergentism Instruments

**Status:** ACTIVE
**Role:** support lane for repository utilities, deterministic renderers, research simulations, and validation scripts that serve the worldview.
**Owns:** `01_SCRIPTS/`, `02_COMPILERS/`, `03_SIMULATIONS/`, `04_DATA_PIPELINES/`, `06_PACKAGES/`, `07_AGENT_OPS/`, `08_AUDIT_ARTIFACTS/`, `90_ARCHIVE/` (local K3), `AGENTS.md`, `CLAUDE.md`, `LICENSE`, `README.md`.
**Must not own:** source doctrine, claim authority, the public site, or any document whose owner is a numbered pillar (01–08, 10).

## Read first

1. `README.md` — pillar README (the existing entry surface).
2. The lane does not own doctrine. Tools compile, lint, render, test, and summarize. Generated output never becomes doctrine merely because a program emitted it.
3. When generated output conflicts with an owner document, repair the owner or generator explicitly and leave a receipt.

## Authority rule

- **Tools are subject to the source owners** (pillars 01–08, 10, 12). A tool that contradicts a canonical source loses.
- **Generated artifacts live in their lane** (e.g. `08_AUDIT_ARTIFACTS/`); receipts are K3-archived.
- **Per-pillar `00_META/` is forbidden** in this lane; governance lives at the root `01_EMERGENTISM/00_META/`.
- **Per-pillar `90_ARCHIVE/` is allowed** for tool-noise tombstones specific to this lane.

## Routing law

A new tool that crosses pillar boundaries must carry an explicit cross-lane receipt naming the source owner. A tool that does not name its owner is not admissible as evidence.
