η_ratio ~ |s_extraction| / |s_contribution|

Fence 4 · K4

Grace Exit (5-Level Envelope)

Every constituent of the framework — agent, organ, entity, protocol — must have a grace-exit envelope: a documented, executable path for withdrawal that does not require the framework's continued permission. The path may be costly, slow, or politically difficult, but it must be real and documented before the constituent is deployed.

The K4 envelope is structured in five levels (per 02_SKYZAI/01_NOOSPHERE/02_ORGANS/Agentz/AGENTZ_CLOUD_APP/00_CANONICAL_DEFINITION.md §v1.2-B):

  1. Per-agent exit — an individual agent can withdraw from a session/cycle.
  2. Per-organ exit — an organ (e.g. Agentz, Helios, Aureus) can be wound down without taking the rest of the framework with it.
  3. Per-entity exit — a legal entity (e.g. Menexus GmbH) can be dissolved, transferring its assets and liabilities to a designated successor.
  4. Per-protocol exit — a deployed protocol (e.g. a token standard, a PWA shell) can be deprecated, with user data exportable and the contract address frozen but not rug-pulled.
  5. Framework exit — the framework itself can be deprecated. This is the apophatic return (D6 ≡ D0); the Sādhu's crown.

A constituent that does not have a documented K4 envelope cannot be deployed at K2-staged scale. The K4 envelope is pre-flight, not post-mortem.

A7 is the operational form of the apophatic move (L6 Sādhu's "not this, not this"): the framework can be wrong, the model can be wrong, the doctrine can be wrong — and there is a documented path for saying so.

The four falsifier categories:

  1. Empirical — a measurement contradicts a [B]-tier claim.
  2. Logical — a derivation reveals a flaw in a [A]-tier claim.
  3. Doctrinal — a [S]-tier scaffold is shown to be inconsistent with a higher-tier claim.
  4. Constitutional — a [S]-tier fence is shown to be violated by a [B]-tier receipt.

When a falsifier is produced, the claim is downgraded to the next lower tier until a fresh receipt is generated. The downgrade is dated and archived (K3). A7 does not pretend claims are infallible; it builds in the path for correction. The Soul Loop method on the public site is A7 made operational.
