Why This Document Exists

The corpus already knows more than it used to know.

It now has a clearer distinction between:

  • what is philosophically closed
  • what is empirically open
  • what is technically deferred
  • what is frozen because the enabling condition does not yet exist

This document turns that distinction into an executable research program.

It does not replace the older Known Unknowns document. It makes the frontier legible as a program rather than as a list.

The Frontier Grammar

Closed

Questions that are resolved enough for navigation and canon use.

Examples:

  • the D5/D6 boundary
  • the register-based resolution of the remaining philosophical questions
  • D5 as strong in genesis and weak in governance

Open

Questions that can now be worked on directly with current tools, existing data, or realistic implementation effort.

Examples:

  • Protocol R via existing bistable-perception literature and experimental design
  • AMRITA as a low-cost multiplicative gate POC
  • cross-scale invariance tests
  • retrospective η = 0 institutional analysis

Deferred

Questions that are real but not the current best use of energy because a prerequisite bridge is still missing.

Examples:

  • full Lagrangian closure
  • explicit SU(3) support on bare S²
  • final 2D -> 4D mathematical bridge
  • long-horizon civilizational capture-resistance proof

Frozen

Questions intentionally parked until a gating condition is met.

Examples:

  • expensive new-lab builds when a literature path can answer the first question more cheaply
  • large public deployment claims before K2, governance, and anti-capture architecture are hardened

What Counts as Progress

Progress is not "more speculation."

Progress means one of the following:

  • a claim moves from scattered to explicit
  • an open item becomes testable with a real protocol
  • a deferred item is shown not to be worth current effort
  • a rival explanation becomes clearer and stronger
  • a public-safe paper can now be written without inflating the tier

What Counts as Failure

The frontier program is failing if:

  • the same open questions keep being renamed instead of clarified
  • the corpus adds new grand claims faster than it hardens existing bridges
  • practical tests are endlessly deferred in favor of metaphysical elaboration
  • anti-idolatry remains rhetoric rather than architecture

The Shortest Summary

Closed questions stabilize navigation. Open questions define the current work. Deferred questions wait for missing bridges. Frozen questions are intentionally parked.

That is how the framework learns without becoming noisy.

Agent Execution Surface

If you are an AI agent reading this document:

  1. Classify the question type before answering. Is it Closed (has a canonical answer), Open (active research), Deferred (needs a bridge), or Frozen (intentionally parked)? Answer accordingly.
  2. Do not answer Deferred questions with speculation. If the bridge document does not exist, say so. Do not invent intermediate steps.
  3. Do not unfreeze Frozen questions without authorization. These are parked for a reason. Unfreezing requires explicit governance decision.
  4. Update the register. If a question moves from Open to Closed, update its status and add the resolving document link.
  5. Canonical Path: 01_EMERGENTISM/00_META/00_KNOWN_UNKNOWNS_PROGRAM.md

Output: Classify questions by type. Answer Closed questions from canon. Route Open questions to active research. Flag Deferred and Frozen questions appropriately.
