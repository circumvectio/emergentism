Evidence tier [B] — file-path evidence

The Pipeline

Source documents live in 01_EMERGENTISM/ as Markdown. A Python generator reads them, applies evidence-tier validation, and emits static HTML. No database. No CMS. No external calls at build time.

Step 1 generate_public_library.py

Scans the doctrine tree. Collects Markdown files with valid front matter. Rejects files missing evidence tiers or publication boundaries.

→
Step 2 build_book.py

Assembles the linear book view — one long scroll through the entire corpus, with provenance arrows on every claim.

→
Step 3 build_atlas_index.py

Builds the cross-reference atlas — every claim linked to its source file, its evidence tier, and its falsifier (if any).

→
Step 4 build_rag_index.py

Generates the RAG chunk index for the AIA engine — embeddings, metadata, citation anchors.

→
Step 5 predeploy_check.py

The gate. Must pass before any deploy. See §3.

[S] These scripts are committed to the repository at the root of 12_PUBLIC_SITE/ (generate_public_library.py, predeploy_check.py, build_book.py). No claim about the pipeline requires trust — only git show.

Evidence tier [A] — enforced by code

The Gate

predeploy_check.py is the last thing that runs before a deploy. It does not ask permission. It fails the build if any of these conditions are violated.

⊘

No external resources

Every page must be self-contained. No CDN fonts, no tracking pixels, no third-party scripts. The site must render correctly on an air-gapped machine. [A]

✓

Evidence tiers on doctrine

Every Markdown file in the public library must declare an evidence tier in its front matter. Files without tiers are rejected. [A]

▣

Publication boundary

Files marked status: draft or publish: false are excluded from the public build. Only K2-signed releases pass. [A]

⌘

Link integrity

All internal links are verified. Broken cross-references fail the build. External links are allowed but marked with a provenance arrow. [B]

◈

Halāhala audit

Superseded claims must carry a Kintsugi seam — a dated tombstone note, not a silent deletion. Missing seams fail the build. [A]

⚡

Build receipt

A signed, resolvable receipt (commit hash, timestamp, file count, tier distribution) is the intended end-state — not yet emitted by the gate. [C] planned.

Evidence tier [S] — functional role map

The Caste Dispatch

The Rosetta system is not AI governance. It is a functional role map — seven jobs that must be filled before any document ships. On this project, all seven roles are executed by language models under human direction. The map is the discipline. The names are optional.

Role Sanskrit name Job What they check
Firewall L1 Caṇḍāla Contradiction isolation Does this claim contradict a settled [A]? Is the tone overconfident?
Truth-Cut L2 Śūdra Evidence-tier discipline Is every claim tiered? Is any [C] presented as fact?
Audit L3 Vaiśya Route verification Do links point to active canon? Are archive files cited as live?
Executor L4 Kṣatriya Build and deploy Does the generator run? Does the gate pass? Is the receipt valid?
Architect L5 Brāhmaṇa Schema and navigation Does the site structure match the canonical tree? Is the map honest?
Compressor L6 Sādhu Archive and tombstone Are superseded claims tombstoned? Is K3 discipline followed?
Witness L7 Ṛṣi Boundary observation Does the public face match the internal discipline? Is the packaging honest?

Evidence tier [B] — command-line evidence

Fork and Verify

You do not need to trust this site. You need to trust your own terminal. Here is the exact sequence to clone, build, and verify.

1

Clone the repository

git clone https://github.com/circumvectio/emergentism.git
cd emergentism/12_PUBLIC_SITE
# Or fork first, then clone your fork
2

Run the gate — zero dependencies

python3 predeploy_check.py
# Pure standard-library Python. No install, no packages to trust.
# 10 gates: link integrity, tier markers, no external resources,
# publication boundary. Exits nonzero on any failure.
3

Regenerate the library (optional)

pip install markdown        # the one dependency
python3 generate_public_library.py --force
# Output lands in _STAGING_COMPASS_RESTRUCTURE/. --force is
# required: the library is frozen with hand-patched tier fixes.
4

Diff against what is served

diff -r . <(curl -s ...)   # or just read the source you cloned
# The page you are reading is in this repo. Nothing is hidden.

[S] The verify step (2) is pure standard-library Python: no package manager, no build chain, nothing to trust beyond your own terminal. The repository goes public at launch. If any command above fails — a 404 on the clone, a missing script, a diff that does not reconcile — that is not a footnote, it is an audit finding, and it is worth reporting.
